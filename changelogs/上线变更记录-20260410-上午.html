一完成需求
  • P0 #7工作流步骤类型扩展
    exec_type 由 2 种扩展为 4 种：智能体执行 / 人工执行 / 人工审核 / 其他（外部AI工具）。 前台步骤卡片新增对应颜色标签，数据库 migration_v9.sql 同步扩展枚举值。
  • P0 #8工作流可见范围交互优化
    旧版需手动输入组织码字符串（如 DEMO,ACME）。改为四模式下拉：全部 / 仅组织用户 / 仅个人用户 / 指定组织（支持搜索+复选框多选）。
  • P0 #9首页联系二维码后台配置
    品牌设置页新增二维码上传区域，支持上传/预览/删除。前台"联系我们"弹窗实时读取。 图片存于 Supabase Storage uploads 桶，需手动创建该桶（Public 开启）。
  • P0 #10列表页多条件筛选
    • 用户管理：按组织/类型/角色/状态/部门筛选，搜索支持手机号/用户名/真实姓名
    • 智能体管理：按类型/分类/状态筛选
    • 工作流管理：按分类/可见范围/状态筛选
    全部支持一键清除筛选。
  • P1 #11批量操作
    用户列表表格新增复选框列，表头支持当页全选；勾选后顶部出现工具栏，可批量启用/禁用。 智能体分配弹窗新增一键全选/全部取消。切换筛选条件时已选项自动清空。
二Bug 修复
  • 修复注册接口 Token 缺字段
    register/route.ts 的 signToken 调用缺少 role 和 userType，TypeScript 编译报错，生产构建失败。已补全。
  • 修复agent_code 含中文时对话/历史失效
    agent_code 为中文时经 URL 编码后，前台、聊天 API、会话 API 三处均未做 decode，导致：
    • 对话页标题显示为 %E6%B5%8B...
    • 聊天 API 找不到智能体，返回 404
    • 历史会话列表加载为空
    三处统一加 decodeURIComponent。
  • 修复多账号分组数据互串
    前台"我的工作任务"分组数据存于固定 key cat_groups_v1，同一浏览器多账号共用同一份数据。 改为 cat_groups_v1_<userId>，按用户隔离。旧数据不迁移，需重建分组。
三过程事故
  • 事故误建 middleware.ts 导致全站 404
    误以为 Next.js 16 需要 middleware.ts 做路由保护，实际该项目已在 proxy.ts 中实现（Next.js 16 专有机制）。两文件共存导致冲突，所有页面返回 404，约 5 分钟后删除 middleware.ts 恢复正常。
