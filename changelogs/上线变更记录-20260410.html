二功能变更（2026-04-09 ~ 04-10）
  • 新增注册入口与登录方式升级
    • 新增 /register 注册页，支持个人用户和组织用户两种注册类型
    • 注册字段：用户名（全局唯一）、真实姓名、手机号、密码（≥8位）、组织码（组织用户必填）
    • 登录页支持手机号或用户名两种登录方式，移除旧版"首次登录自动建号"逻辑
  • 新增用户类型与角色体系重构
    • 用户分为个人用户（personal）和组织用户（organization）两种类型
    • 新增四级系统角色：普通用户 / 组织管理员 / 系统管理员 / 超级管理员
    • 管理后台用户列表新增用户类型列、系统角色列，支持按类型/角色筛选，可点击角色标签直接修改
    • 前台首页顶栏实时展示当前用户角色标签（修改后无需重新登录即时生效）
  • 新增固定三级组织结构
    • 新建部门（departments）和小组（teams）两级结构，挂载在租户下
    • 租户管理页支持展开组织树，可内联增删改部门和小组
    • 用户管理页新增"分配部门"功能，可将组织用户分配至指定部门/小组
  • 优化权限与可见性模型升级
    智能体可见范围控制粒度从"按组织"升级为多维度支持：
    • 全部用户 / 指定组织 / 指定部门 / 指定小组 / 按用户类型 / 指定用户
    • 智能体管理页新增权限设置弹窗，支持增删权限条目，含标签展示
  • 优化组织码规则升级
    组织码限制为 4~8 位纯英文字母，不区分大小写，展示时统一转大写；前端实时过滤非法字符，后端同步校验。
  • 新增工作流步骤类型扩展
    步骤执行类型由 2 种扩展为 4 种：智能体执行 / 人工执行 / 人工审核（紫色标签）/ 其他（外部 AI 工具）（灰色标签），前台步骤卡片可视化区分四种类型。
  • 优化工作流可见范围设置交互优化
    • 旧版为手动输入组织码字符串（如 DEMO,ACME），体验差
    • 新版改为四模式下拉选择：全部 / 仅组织用户 / 仅个人用户 / 指定组织（含搜索+复选框）
  • 新增首页联系二维码后台配置
    • 品牌设置页新增"二维码上传"区域，支持上传、预览、删除联系二维码
    • 前台首页"联系我们"弹窗实时读取后台配置的二维码图片及说明文字
    • 文件存储于 Supabase Storage uploads 桶，路径 logos/contact-qr.*
  • 新增列表页多条件筛选
    • 用户管理：按组织、用户类型、角色、状态、部门筛选，搜索支持手机号/用户名/真实姓名
    • 智能体管理：按类型、分类、状态筛选
    • 工作流管理：按分类、可见范围、状态筛选
    • 全部列表页均支持"清除筛选"一键重置
  • 新增批量操作
    • 用户管理：表格新增复选框列，支持当页全选；勾选后显示工具栏，可批量启用/禁用
    • 智能体管理：分类分配弹窗和展示配置弹窗均新增"一键全选"/"全部取消"按钮
  • 优化全站术语替换
    全站"企业"统一改为"组织"：企业码 → 组织码、企业管理 → 组织管理、企业分配 → 组织分配；覆盖侧边栏导航、控制台、API 错误提示及代码注释。
三Bug 修复
  • 修复注册接口 Token 字段缺失
    app/api/auth/register/route.ts 中 signToken 调用缺少 role 和 userType 字段，导致 TypeScript 编译错误，生产构建失败。已补全两个字段。
  • 修复智能体 agent_code 含中文时 URL 编码问题
    当 agent_code 包含中文字符时，URL 经浏览器编码后 params.id 返回编码态字符串，导致：
    • 对话页面智能体名称显示为 %E6%B5%8B...
    • 聊天 API 查不到智能体，返回"智能体不存在"
    • 历史会话列表加载失败
    已在前台页面、聊天 API、会话 API 三处统一添加 decodeURIComponent。
    建议：agent_code 字段使用英文+数字，避免此类问题。
  • 修复前台工作任务分组数据多账号共用问题
    首页左侧"我的工作任务"分组使用固定 key cat_groups_v1 存储于 localStorage，导致同一浏览器下多个账号共享分组数据，账号切换后会看到其他账号的分组。 已将 key 改为按用户 ID 隔离（cat_groups_v1_<userId>），各账号分组数据互不影响。
    注意：旧版已建的分组不会自动迁移，需用户重新创建。
  • 修复proxy.ts 与 middleware.ts 冲突
    误创建 middleware.ts 导致与 Next.js 16 的 proxy.ts 路由保护机制冲突，造成所有页面返回 404。已删除 middleware.ts，恢复正常。
