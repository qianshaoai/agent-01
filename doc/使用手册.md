## 一、系统概述

本系统是一个**多租户 AI 智能体统一门户**，允许管理员将来自不同 AI 平台的智能体（如扣子、Dify、智谱清言、腾讯元器等）统一接入，并按企业（租户）分配使用权限。

**核心角色：**

| 角色 | 入口 | 职责 |
|------|------|------|
| 超级管理员 | `/admin` | 管理企业、智能体、用户、公告、查看统计 |
| 企业用户 | `/login` | 使用分配给本企业的智能体进行对话 |
| 个人用户 | `/login`（企业码留空） | 使用系统内智能体，独立账号空间 |

## 二、部署与初始化

### 2.1 环境要求

- Node.js 18+
- npm 9+
- Supabase 项目（免费套餐即可）
- 可选：Vercel 账号（用于生产部署）

### 2.2 Supabase 数据库配置

1. 登录 [https://supabase.com/dashboard](https://supabase.com/dashboard)，新建项目

2. 记录以下三个信息：

   | 信息 | 位置 |
   |------|------|
   | Project URL | Settings → API → Project URL |
   | anon key | Settings → API → Project API keys → anon public |
   | service_role key | Settings → API → Project API keys → service_role secret |

3. 进入 **SQL Editor**，依次执行：
   - `supabase/schema.sql`（建表和初始化数据）
   - `supabase/rpc.sql`（配额原子扣减函数）

4. 进入 **Storage**，创建名为 `uploads` 的公开 Bucket（用于文件上传）

执行完成后系统内置：
- 管理员账号：`admin` / 密码 `admin`（**生产环境必须修改**）
- 测试企业：企业码 `DEMO` / 企业密码 `demo123`，配额 500 次

### 2.3 本地开发启动

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量
cp .env.local.example .env.local
# 编辑 .env.local，填入 Supabase 信息和 JWT 密钥

# 3. 生成 JWT_SECRET（至少 32 位）
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

# 4. 启动开发服务器
npm run dev
# 访问 http://localhost:3000
```

### 2.4 Vercel 生产部署

1. 将代码推送到 GitHub
2. 在 Vercel 导入项目，框架选择 **Next.js**
3. 在 Vercel → Settings → Environment Variables 中添加：

```
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
JWT_SECRET
# 可选，启用语音识别：
VOLCENGINE_APP_ID
VOLCENGINE_ACCESS_TOKEN
```

4. 点击 Deploy 完成部署

## 三、管理员操作指南

管理员入口：`/admin`，默认账号 `admin` / `admin`

### 3.1 控制台（Dashboard）

登录后进入控制台，显示系统整体概况：
- 企业数量、用户数量、智能体数量
- 近期使用趋势图表

### 3.2 企业码管理

路径：**管理后台 → 企业码管理**

#### 新增企业

点击右上角「新增企业码」，填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| 企业码 | 全局唯一，自动转大写，创建后不可修改 | `COMPANY2024` |
| 企业名称 | 显示名称 | 前哨科技有限公司 |
| 企业初始密码 | 该企业所有用户首次登录的共用密码 | `demo123` |
| 总配额（次数） | 该企业可使用的 AI 对话总次数 | `500` |
| 到期日 | 超过此日期后该企业用户无法使用 | `2026-12-31` |

#### 编辑企业

点击列表中的 ✏️ 图标，可修改名称、密码、配额和到期日（企业码不可修改）。

#### 启用 / 禁用企业

点击 🚫 图标可立即禁用该企业（用户无法登录）；再次点击恢复启用。

#### 配额说明

配额进度条颜色：
- **蓝色**：使用正常（< 80%）
- **橙色**：临近用尽（≥ 80%）
- **红色**：已耗尽（100%），该企业用户将无法发起对话

> 配额耗尽或到期后，需在此页面修改配额或延长到期日方可恢复使用。

---

### 3.3 智能体管理

路径：**管理后台 → 智能体管理**

#### Tab 切换

- **智能体列表**：查看、添加、配置所有智能体
- **分类管理**：管理智能体分类标签

#### 新增智能体

点击右上角「新增智能体」，填写：

| 字段 | 说明 |
|------|------|
| 智能体编号（ID） | 全局唯一，如 `AGT-001`，创建后不可修改 |
| 名称 | 用户看到的显示名称 |
| 简介 | 简短描述该智能体的用途 |
| 分类 | 选择所属分类（可不选） |
| 智能体类型 | 见下方说明 |
| 对接平台 | AI 后端平台（仅对话型） |

**智能体类型：**

| 类型 | 说明 |
|------|------|
| 站内对话型 | 用户在平台内进行多轮对话，历史记录保存在系统中 |
| 外链跳转型 | 点击后跳转到外部 URL，不在系统内对话 |

#### 配置 API（🔑 图标）

创建智能体后，点击列表中的 🔑 图标配置接口信息：

| 字段 | 说明 |
|------|------|
| API Endpoint | 接口地址，留空使用各平台默认地址 |
| API Key | 平台 API 密钥（加密存储，只写不读） |
| 模型参数（JSON） | 额外参数，格式因平台而异（见第五节） |

#### 企业分配（⚙️ 图标）

点击 ⚙️ 图标，勾选哪些企业可以使用该智能体。

> 未分配给任何企业的智能体，仅管理员可见，个人用户（无企业码）也无法使用。

#### 分类展示配置（⊞ 图标）

控制该智能体在各分类下的显示状态：

| 操作 | 说明 |
|------|------|
| 手动添加（+ 图标） | 将智能体添加到该分类的展示页 |
| 隐藏（👁 图标） | 在该分类中隐藏此智能体（优先级最高） |

#### 分类管理

在「分类管理」Tab 中：
- 输入名称后按 Enter 或点击「添加」新建分类
- 点击 ✏️ 图标内联编辑分类名
- 点击 🏢 图标配置哪些企业可以看到该分类

## 四、普通用户操作指南

### 4.1 登录

访问 `/login` 页面：

| 字段 | 说明 |
|------|------|
| 手机号 | 任意手机号格式 |
| 企业码 | 由管理员提供；个人使用可留空 |
| 密码 | 企业初始密码（首次登录后强制修改） |

**首次登录流程：**
1. 输入手机号、企业码、初始密码（由管理员告知）
2. 系统自动创建账号并登录
3. 弹出修改密码弹窗，**必须修改后才能继续使用**

**个人用户：**
- 企业码留空，系统使用 `PERSONAL` 作为个人空间
- 首次登录默认密码为 `000000`，登录后同样需要修改密码

---

### 4.2 智能体广场（首页）

登录后进入首页，按分类展示所有可用智能体卡片：
- 每张卡片显示智能体名称和简介
- 点击卡片进入对话页面
- 外链型智能体点击后在新标签页打开目标网址

> 仅显示管理员分配给本企业的智能体，不同企业看到的内容可能不同。

---

### 4.3 与智能体对话

点击智能体卡片，进入聊天界面：

**基本操作：**
- 在底部输入框输入问题，按 Enter 或点击发送按钮
- AI 回复以流式输出（逐字显示）
- 系统自动保存对话历史，每次对话携带最近 20 轮（40 条消息）作为上下文

**文件上传：**
- 点击输入框旁的上传图标选择文件
- 系统提取文件文本内容并附加到当前消息发送给 AI
- 上传文件存储在云端，可在同一对话中多次引用

**语音输入：**（需管理员配置语音识别）
- 点击麦克风图标开始录音
- 录音结束后自动识别为文字填入输入框

**对话历史：**
- 左侧边栏（或导航区）显示历史对话列表
- 点击可切换到历史对话继续聊天
- 新开一次对话会自动创建新的会话记录

---

### 4.4 账户设置

路径：**右上角头像 → 设置** 或直接访问 `/settings`

- 修改登录密码
- 查看账号信息（手机号、所属企业）
- 注销账号

## 五、AI 平台接入说明

管理员在「智能体管理 → API 配置」中填写各平台参数。

### 5.1 扣子（Coze）

| 字段 | 内容 |
|------|------|
| 平台 | `coze` |
| API Endpoint | 留空（默认 `https://api.coze.cn/v3/chat`） |
| API Key | 扣子平台的 Personal Access Token |
| 模型参数 | `{"bot_id": "你的Bot ID"}` |

获取 Bot ID：在扣子平台发布 Bot 后，从浏览器地址栏复制。

---

### 5.2 Dify

| 字段 | 内容 |
|------|------|
| 平台 | `dify` |
| API Endpoint | 留空（默认 `https://api.dify.ai/v1/chat-messages`）或填私有部署地址 |
| API Key | Dify 应用的 API Key |
| 模型参数 | `{"temperature": 0.7}` |

---

### 5.3 智谱清言（Qingyan）

> 详见 `doc/智谱清言智能体配置说明.md`

| 字段 | 内容 |
|------|------|
| 平台 | `qingyan` |
| API Endpoint | 留空（默认 `https://chatglm.cn/chatglm/assistant-api/v1`） |
| API Key | 清言开发者面板的 Key（如 `4e981627c1398d03`） |
| 模型参数 | `{"api_secret": "你的Secret", "assistant_id": "智能体ID"}` |

**注意事项：**
- Secret 仅在创建时显示一次，请立即保存
- 每个账号每天最多调用 5000 次
- 最多创建 5 个 API Key

---

### 5.4 腾讯元器（Yuanqi）

| 字段 | 内容 |
|------|------|
| 平台 | `yuanqi` |
| API Endpoint | 留空（默认腾讯元器地址） |
| API Key | 元器平台 API Key |
| 模型参数 | `{"assistant_id": "你的智能体ID"}` |

---

### 5.5 OpenAI 兼容接口

适用于任何兼容 OpenAI Chat Completion 格式的服务（如 DeepSeek、Moonshot、本地 Ollama 等）：

| 字段 | 内容 |
|------|------|
| 平台 | `openai`（或任意其他字符串） |
| API Endpoint | 完整接口地址，如 `https://api.deepseek.com/v1/chat/completions` |
| API Key | 对应平台的 API Key |
| 模型参数 | `{"model": "deepseek-chat", "temperature": 0.7, "max_tokens": 2000}` |

## 六、账号体系说明

### 登录方式对比

| | 企业用户 | 个人用户 |
|--|---------|---------|
| 手机号 | 任意格式 | 任意格式 |
| 企业码 | 必填（由管理员提供） | 留空 |
| 初始密码 | 企业初始密码 | `000000` |
| 可见智能体 | 分配给本企业的智能体 | 暂无（需管理员分配） |
| 配额 | 归属所在企业配额 | 无限制（暂无配额控制） |

### 密码安全

- 所有密码均使用 bcrypt 加密存储，系统无法查看明文密码
- 如企业用户忘记密码，管理员可在「企业码管理」中重置企业初始密码，用户使用新初始密码登录后再次设置个人密码
- 管理员密码如需重置，需在 Supabase SQL Editor 中直接更新 `admins` 表的 `pwd_hash` 字段

## 七、常见问题

**Q：用户登录时提示"企业码不存在"或"密码错误"？**
> 检查企业码是否正确（注意大小写，系统自动转大写）；确认管理员已创建该企业码且企业处于启用状态。

**Q：用户登录成功但看不到任何智能体？**
> 管理员需在「智能体管理 → 企业分配」中将智能体分配给该企业。

**Q：发起对话时提示"配额不足"或"服务已到期"？**
> 联系管理员在「企业码管理」中增加配额或延长到期日。

**Q：AI 回复报错"API Key 无效"？**
> 管理员重新进入「智能体管理 → API 配置」，重新填写并保存 API Key。

**Q：智谱清言调用失败？**
> 检查模型参数中的 `api_secret` 和 `assistant_id` 是否正确；确认智能体已在清言平台**发布**；检查当日调用次数是否已达 5000 次上限。

**Q：文件上传失败？**
> 确认 Supabase Storage 中已创建名为 `uploads` 的公开 Bucket；检查文件大小是否超限。

**Q：语音输入按钮不可用？**
> 该功能需要管理员在环境变量中配置 `VOLCENGINE_APP_ID` 和 `VOLCENGINE_ACCESS_TOKEN`。

**Q：如何修改管理员密码？**
> 在 Supabase Dashboard → SQL Editor 中执行：
> ```sql
> UPDATE admins SET pwd_hash = '$2b$10$...' WHERE username = 'admin';
> ```
> `pwd_hash` 需使用 bcrypt 加密后的字符串（saltRounds=10）。
