# 需求差距分析报告
> 基于《260408 智能体舱优化需求文档（补丁）V2.0》对照现有代码进行分析
> 分析日期：2026-04-09

---

## P0 — 本轮必须完成（核心功能缺失）

### ✅ 1. 术语全量替换（已完成 2026-04-09）
- 全站所有"企业码"→"组织码"、"企业管理"→"组织管理"、"企业分配"→"组织分配"
- 覆盖：侧边栏导航、控制台、租户管理、智能体管理、公告管理、用量看板、操作日志、工作流管理、用户设置页、所有 API 错误提示文案及代码注释

---

### ✅ 2. 组织码规则升级（已完成 2026-04-09）
- 只允许 4~8 位英文字母，大小写不敏感，展示统一大写
- 覆盖：注册页前端实时过滤 + 注册 API 后端校验 + 管理员新增组织前端 + 租户 API 后端校验

---

### ✅ 3. 用户类型与角色体系重构（已完成 2026-04-09）
- users 表新增 `user_type`（personal / organization）和 `role`（super_admin / system_admin / org_admin / user）字段
- 用户管理页新增用户类型、系统角色两列，支持按类型/角色筛选
- 管理员可点击角色标签直接修改用户角色

---

### ✅ 4. 固定三级组织结构（已完成 2026-04-09）
- 新建 `departments`（部门）、`teams`（小组）表，users 表新增 `dept_id`、`team_id` 字段
- 租户管理页支持展开组织结构，可内联增删改部门和小组
- 用户管理页新增"分配部门"按钮，可将组织用户分配至指定部门/小组

---

### ✅ 5. 注册入口与登录方式升级（已完成 2026-04-09）
- 新增 `/register` 注册页，支持个人/组织用户类型切换
- 注册字段：用户名（全局唯一）、真实姓名、手机号、密码（≥8位）、组织码（组织用户）
- 登录页简化为账号+密码，支持手机号或用户名两种方式登录
- 移除登录自动建号逻辑，新用户必须通过注册页注册
- users 表新增 `username`、`real_name` 字段，管理页用户列展示真实姓名和 @用户名

---

### ✅ 6. 权限与可见性模型升级（已完成 2026-04-09）
- 新建 `resource_permissions` 统一权限表，替换 `tenant_agents`
- 支持按：全部用户 / 组织 / 部门 / 小组 / 用户类型 / 指定用户 控制智能体可见范围
- 智能体管理页权限设置弹窗（增删权限条目，含标签展示）
- 迁移文件：`supabase/migration_v8.sql`

---

### ✅ 7. 工作流步骤类型扩展（已完成 2026-04-10）
- **现状**：只有"智能体执行"和"人工执行"两种类型
- **需求**：扩展为四种：智能体执行 / 人工执行 / 人工审核 / 其他（外部 AI 工具）
- **涉及**：数据库 `exec_type` 字段枚举值扩展 + 前台步骤标签视觉区分

---

### ✅ 8. 工作流可见权限操作优化（已完成 2026-04-10）
- **现状**：只能手动输入组织码字符串（如 `DEMO,ACME`），体验差
- **需求**：改为下拉选择组织，支持搜索，不再靠手输组织码

---

### ✅ 9. 首页联系二维码后台配置（已完成 2026-04-10）
- **现状**：首页有二维码展示位，但后台没有上传入口
- **需求**：在品牌设置中增加二维码上传、替换、删除功能
- **缺失**：`system_settings` 表缺少二维码相关配置项，品牌设置页缺少上传区域

---

### ✅ 10. 列表页筛选功能补齐（已完成 2026-04-10）
- **现状**：大多数列表页只有搜索框，没有多条件筛选
- **需求**：
  - 用户管理：按组织、用户类型、角色、状态、部门、分组筛选
  - 工作流管理：按分类、可见范围、组织、状态筛选
  - 智能体管理：按类型、分类、可见范围、分配组织、状态筛选

---

## P1 — 本轮建议完成

### ✅ 11. 批量操作（已完成 2026-04-10）
- **现状**：全部是逐条操作，没有批量能力
- **需求**：
  - 智能体分类分配弹窗：一键全选 / 取消
  - 智能体组织分配弹窗：一键全选 / 全部取消
  - 用户列表：批量启用 / 禁用

---

### ✅ 12. 工作流后台分类管理（已完成 2026-04-10）
- **现状**：工作流只有一个自由文本"分类标签"字段，无结构化分类
- **需求**：建立独立的工作流分类管理模块，可按部门、业务线、场景等分类

---

### ✅ 13. 组织删除逻辑（已完成 2026-04-10）
- **现状**：只能禁用组织，不能删除
- **需求**：
  - 有关联数据（用户、工作流、智能体等）时禁止删除并给出提示
  - 无关联数据时允许删除，删除前二次确认

---

### 14. 超级管理员删除用户
- **现状**：只能禁用用户，不能删除
- **需求**：超级管理员可软删除用户，删除前二次确认，处理关联数据避免数据孤儿

---

### 15. 用户分组（权限分组）
- **现状**：完全没有此功能
- **需求**：后台新增用户分组管理，分组作为权限标签，用于控制资源可见范围（与前台"我的分组"严格区分）

---

## P2 — 后续迭代

### 16. UI / UX 中度重设计
- 字号偏小，阅读不舒适
- 信息层级不强，重点不够突出
- 图标风格不统一
- 后台操作效率偏低，不像高效管理台
- 前台工作流区纵向拖沓感强
- 智能体展示模块折叠栏过弱
- **方向**：中度优化，不推翻重做，重点提升可读性、操作效率、信息层级清晰度
