## 桌面《智能体舱平台功能清单》改写为用户视角版

### 背景

桌面上的 `智能体舱平台功能清单.docx` 是研发内部视角的版本：

- 表格 8 列，含「优先级」P0/P1/P2 列
- 大量内部代号：`master / devB+ / devA`、`5.13up Fix 1/2/3`、`P5/P6/P7 待办`
- 大量技术词汇：`endpoint`、`api_key`、`assistant_id`、`Bot ID`、`/v3/chat SSE API`、`/v1/chat-messages`、`workflow_session`、`_enc 字段`、`JWT_SECRET`、`HttpOnly cookie`、`TTL`、`force_relogin_at`、`VOLCENGINE_APP_ID` 等
- 花哨符号：中点 `·`、箭头 `→`、斜杠 `/` 当连词、省略号 `……`、括号内英文标注

用户（小B 视角）反馈"容易看花眼"、"太技术"，要求改成"从用户角度、一眼看懂"。

### 决策

- 落地方式：**直接覆盖原文件**（原研发版本不保留；如后续需要技术版本另行整理）
- 优先级列：**整列删除**（普通用户看不懂 P0/P1/P2）

### 执行

新表结构 **7 列**：

| 列名 | 原列对应 | 改写要点 |
|---|---|---|
| 模块 | 模块 | 保留 |
| 功能 | 功能点 | 去英文标注（"清言（Qingyan）" → "清言"） |
| 这个功能能做什么 | 用户能看到/使用什么 | 列名更白话 |
| 状态 | 当前状态 | 去掉版本号（"已完成（5.13up Fix 2）" → "已上线"） |
| 谁能用 | 支持范围 | 列名更白话 |
| 怎么用 | 使用方式 | 箭头 → 改为"进入XX，点XX" |
| 注意事项 | 当前问题或限制 | 删技术细节，留对用户有意义的限制 |
| ~~优先级~~ | ~~优先级~~ | **删除** |

文案改写原则：

- 凡接口路径（`/v3/chat`、`/v1/chat-messages` 等）一律删除
- 凡变量/字段名（`endpoint`、`assistant_id`、`workflow_session` 等）翻译成中文（"接口地址"、"智能体编号"、"系统自动隔离"）
- 凡内部代号（`5.13up Fix 1/2/3`、`master/devB+/devA`、`P5 待办` 等）一律删除
- 凡缩写（`SSE`、`TTL`、`JWT`、`CRUD`、`UX`）改为中文描述
  - 例：`JWT 30 天 TTL + 加密 cookie` → `登录态保持 30 天`，描述改为"登录一次后 30 天内免重新登录"
  - 例：`workflow_session 隔离` → "对话历史互不干扰"
- 花哨符号：中点 `·` 全部去掉；箭头 `→` 改为"进入…，点…"；斜杠 `/` 当连词改为顿号

视觉重做：

- 标题居中、字号 20pt、加粗
- 副标题"截至 2026 年 5 月 14 日"（去掉原"覆盖 master/devB+/devA 所有已合并分支"）
- 表头蓝底白字、加粗
- 数据行斑马纹（白色 + 浅蓝灰 #F5F7FA 交替）
- 状态列字体加粗 + 绿色（视觉上的"已上线"提示）
- 字体统一微软雅黑

### 项数校正

原文件末尾写"共 61 项"，实际表格项 60 个（grep `^P\d` 计数）。新版本末尾改为"共 60 项已实现功能"。
