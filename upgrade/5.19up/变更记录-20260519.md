## 搭建器发布可见范围接入 resource_permissions

### 背景

搭建器「发布设置 → 可见范围」原来只把选择存进 `visibility_config`，发布接口不读它、
也不写 `resource_permissions` —— 发布的智能体没有权限规则、前台组织用户谁都看不见。
本次把发布与权限打通：发布时按「可见范围 + 发布者角色」写 agent 的 `resource_permissions`。

### 权限口径（用户拍板）

- **super_admin / system_admin**：可发「全平台可见」/「指定组织可见」（组织多选）/
  「暂不公开」。
- **org_admin**：最多只能发「本组织可见」/「暂不公开」；不能发「全平台可见」、不能
  指定其他组织。
- 本期不动个人用户 `/api/agents` 的可见性逻辑。

### 改动

**`supabase/rpc.sql`** —— 新增 RPC `set_agent_permissions(p_agent_id, p_perms)`：
plpgsql 单事务，先删该 agent 全部 `resource_permissions` 行、再按 `p_perms`（JSONB
数组）全量写入；`INSERT ... ON CONFLICT DO NOTHING`；文件末 `NOTIFY pgrst,
'reload schema'`。**⚠ 需在目标库 SQL Editor 执行该函数。**

**`app/api/admin/agent-drafts/[id]/publish/route.ts`**：
- 去掉原「禁止 org_admin 发布」拦截 —— 改为 org_admin 可发布、但后端强制最大可见
  范围为本组织。
- 可见范围校验放在任何 DB 写入**之前**（旧值 `custom` / 未知 → 降级 `owner_only`）：
  - org_admin：无 `tenantCode` → 403；`visible_to=all` → 403；`visible_to=org` →
    scope 未传则兜底归一化为 `[admin.tenantCode]`，传了则必须恰为 `[admin.tenantCode]`、
    含其他组织码 → 403。
  - super / system：`visible_to=org` → scope 去重、非空（空 → 422）、每个组织码
    必须在 `tenants` 表存在（无效 → 422）。
- agent upsert 之后，按 `visible_to` 组装 perms 调 `set_agent_permissions` RPC：
  `all` → 1 条 `scope=all`；`org` → 每组织码 1 条 `scope=org`；`owner_only` → 不写。
- RPC 失败 → 发布返回 500，**不更新 draft 状态为 published、不写成功审计**（幂等、
  可重试）。

**`app/admin/agent-builder/[id]/page.tsx`**：
- `load()` 加拉 `GET /api/admin/me` 取当前管理员角色。
- 可见范围下拉按角色：super / system 显示「全平台可见 / 指定组织可见 / 暂不公开」；
  org_admin 显示「本组织可见 / 暂不公开」（不展示「全平台」「指定组织多选」）。
- 「指定组织可见」的组织多选 + `fetchAllTenants`（分页循环拉全量）**仅 super / system**；
  org_admin 不拉 tenants、不传 scope（后端兜底本组织）。
- super / system 选「指定组织可见」但未勾任何组织 → 发布按钮拦下 toast。
- 重发布弹窗提示：发布会按搭建器「可见范围」覆盖现有权限。
- 旧草稿 `visible_to="custom"` 加载时降级「暂不公开」。

### 验证
