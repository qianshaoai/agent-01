## 搭建器发布可见范围接入 resource_permissions

### 背景

搭建器「发布设置 → 可见范围」原来只把选择存进 `visibility_config`，**发布接口完全
不读它、也不写 `resource_permissions`** —— 发布出的智能体没有权限规则、前台组织用户
谁都看不见。本次补上这半截：发布时按可见范围写 agent 的 `resource_permissions`。

### 改动

**`supabase/rpc.sql`** —— 新增 RPC `set_agent_permissions(p_agent_id, p_perms)`：
plpgsql 单事务，先删该 agent 全部 `resource_permissions` 行、再按 `p_perms`（JSONB
数组）全量写入，要么全成要么全不动。**⚠ 需在目标库 SQL Editor 执行该函数。**

**`app/api/admin/agent-drafts/[id]/publish/route.ts`**：
- 校验 `visibility_config` / `scope` —— 放在任何 DB 写入**之前**（与 name / provider
  校验同段）：`visible_to="org"` 时 `scope` 去重、非空，每个组织码必须在 `tenants`
  表存在；不过即 422，此时尚无任何 DB 写。
- 旧值 `visible_to="custom"` / 未知值 → 降级 `owner_only`（安全侧）。
- agent upsert 之后，按 `visible_to` 组装 perms 调 `set_agent_permissions` RPC：
  `all` → 1 条 `scope=all`；`org` → 每组织码 1 条 `scope=org`；`owner_only` → 不写。
- RPC 失败 → 发布返回 500，且**不把 draft 标 `published`**（幂等、可重试）。

**`app/admin/agent-builder/[id]/page.tsx`**：
- 「可见范围」下拉改 3 档：**全平台可见 / 指定组织可见 / 暂不公开**（去掉「自定义
  范围」「本组织可见」）。「仅本人可见」→「暂不公开（仅自己测试，前台不可见）」，
  名实一致。
- 选「指定组织可见」→ 下方组织多选（复选列表）；组织列表 `fetchAllTenants` 分页
  循环拉全量（不受 tenants 接口默认 100 上限）。
- `VisibilityConfig.scope` 类型 `unknown[]` → `string[]`；`visible_to` 去掉 `"custom"`。
- 加载旧草稿时 `visible_to="custom"` 降级 `owner_only`。
- 重发布弹窗加提示：发布会按可见范围**覆盖**「权限设置」里的现有权限。

### 决策（方案 D1~D4）

- **D1**：org_admin 不能用搭建器（`agent-drafts` 列表/创建/发布三接口都 super /
  system 专属，5.14up 起即如此）→「本组织 / 指定组织」不分角色，统一「指定组织可见」
  多选。
- **D2**：不加 user_type 档。**D3**：重发布覆盖手工权限，可接受 + 弹窗提示。
- **D4**：本功能不碰 `/api/agents`；个人用户绕过权限表（看所有 enabled agent）是
  已知现状，「个人用户纳入权限约束」拆为产品另议的独立项，不阻塞本功能。

### 验证

- `ci:typecheck` / `ci:lint` / `ci:build` 通过。
- 待用户验收（dev server 已热更新）：
  - 3 档分别发布 → 查该 agent 的 `resource_permissions` 行。
  - 「指定组织」多选；选 0 个 / 无效组织码被 422 拦。
  - 重发布 → 旧权限被覆盖、弹窗有提示。
  - 旧 `visible_to="custom"` 草稿打开降级「暂不公开」、不报错。
