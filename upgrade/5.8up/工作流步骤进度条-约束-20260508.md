# 5.8up 约束文档：工作流步骤进度条

**日期**：2026-05-08

---

## 一、步骤跳转规则

| 场景 | 行为 | 说明 |
|---|---|---|
| 点击「进入下一步」 | 弹出确认对话框，确认后顺序前进 | 无论当前步是否发过消息，均弹提示（见§七） |
| 点击已完成的步骤 | 允许跳回，**可以继续发消息** | 回头补充信息不受限 |
| 点击未到达的步骤 | 禁止 | 灰色不可点 |
| 刷新页面 | URL 参数还在，步骤条正常恢复 | 进度不丢失，但「已完成」状态靠前端 state 维护，刷新后清空（见§三） |
| 直接输入 URL 跳到某步 | 允许，但无上下文注入 | `from` 参数为空则不注入，智能体正常工作 |

---

## 二、上下文传递规则

| 项目 | 规则 |
|---|---|
| 注入时机 | 仅在该步骤「第一条用户消息」发出时注入，后续消息不重复注入 |
| 注入方式 | 作为 `system` 角色消息，标题为「【上一步工作记录】」，注入在 agent 系统提示之后、对话历史之前 |
| 注入内容 | 优先使用该会话已有的摘要（`conversations.summary_text`）；若无摘要，取该会话最近 20 条消息格式化为纯文本 |
| 消息长度限制 | 上下文文本超过 **2000 字符**时，截断并末尾加「…（内容过长已截断）」 |
| 注入失败处理 | 拉取上一步会话失败（网络/权限问题）→ 静默忽略，不阻断对话，不提示用户 |
| 入库 | 不入库。注入的 system 消息仅存在于本次 API 调用，不写入 messages 表 |

---

## 三、进度持久化规则

| 项目 | 规则 |
|---|---|
| 步骤进度 | 靠 URL 参数（`step`）维护，刷新不丢 |
| 已完成状态（绿勾）| 靠前端 state 维护：进入某步时把 step-1 标为已完成；**刷新后清空**（已完成步退回灰色可点状态，但不影响功能） |
| 会话 ID | `conversationId` 由聊天逻辑正常维护，`from` 参数只在跳步时一次性使用 |
| 跨设备 | 不支持。工作流进度不同步到数据库，换设备/浏览器从头开始 |

---

## 四、显示规则

| 场景 | 进度条是否显示 |
|---|---|
| URL 含 `wf` 参数，且能成功拉取步骤列表 | 显示 |
| URL 无 `wf` 参数（普通智能体入口） | 不显示，界面与现在完全一致 |
| `wf` 参数对应的工作流不存在或已删除 | 不显示，静默降级为普通聊天 |
| 工作流只有 1 个步骤 | 不显示进度条（单步没意义），但「完成」按钮仍显示 |

---

## 五、权限规则

| 项目 | 规则 |
|---|---|
| 谁能进入工作流步骤聊天 | 与普通智能体一致，取决于该智能体的可见范围（resource_permissions） |
| 步骤 API（`/api/workflows/[id]/steps`） | 需要用户登录；org_admin 按原有工作流可见范围过滤 |
| 跨工作流 | 不允许（URL 篡改 `wf` 参数指向另一工作流，步骤列表重新加载，`from` 参数沿用，但注入的上下文可能来自无关会话，属于用户自行操作，不做额外校验） |

---

## 六、不在本次做的事

| 项目 | 原因 |
|---|---|
| 工作流步骤完成状态持久化到 DB | 增加复杂度，当前需求不要求多端同步 |
| 已完成步骤不可继续对话 | 用户可能需要回头补充，限制太死 |
| 工作流整体完成后的汇总报告 | 超出本期范围，可作为 5.9up 候选 |
| 移动端适应（步骤条折叠/横滑） | 平台以 PC 为主，本期不做响应式适配 |
| 步骤之间双向传递（后步传给前步） | 当前只支持单向：前步 → 后步 |

---

## 七、切换确认对话框规则

每次点击「进入下一步」或「完成工作流」时，必须弹出确认对话框，**无论当前步骤是否发过消息**。

**对话框内容**：

- 非最后一步：
  - 标题：「即将进入下一步」
  - 正文：「您即将离开【当前步骤名】，进入【下一步骤名】。上一步的对话内容将自动传递给下一个智能体。」
  - 按钮：「继续」（确认）/ 「取消」

- 最后一步：
  - 标题：「完成工作流」
  - 正文：「恭喜！您已完成【工作流名称】的所有步骤。」
  - 按钮：「返回首页」/ 「取消」

## 八、进度条展示规则（用户拍板版）

- **顶部大标题**：工作流名称（如「客户需求调研流程」）
- **步骤小标题**：对应智能体名称（如「需求分析助手」）
- **步骤序号**：① ② ③ … 自动生成

示意：

```
工作流：客户需求调研流程
─────────────────────────────────────────────────
  ① 需求分析助手 ✓  →  ② 方案设计助手（当前）  →  ③ 评审确认助手
─────────────────────────────────────────────────
```

## 九、所有拍板结论汇总

| 问题 | 结论 |
|---|---|
| 回看已完成步骤时能发消息吗 | 能，不限制 |
| 「完成」后的行为 | 弹完成弹窗，含「返回首页」按钮 |
| 步骤名称显示什么 | 大标题=工作流名称，小标题=智能体名称 |
| 没聊就切换怎么处理 | 无论是否聊过，切换前均弹确认对话框 |
