# 5.8up 方案：工作流步骤进度条 + 跨步骤上下文传递

**日期**：2026-05-08  
**分支**：`devB`  
**需求来源**：用户口述——工作流涉及多个智能体时，希望在聊天界面内直接切换，无需退出重进；并希望下一个智能体能读到上一步的对话内容。

---

## 一、功能描述

用户从首页进入一个工作流时：

1. 聊天界面顶部出现步骤进度条，显示该工作流的所有步骤（智能体名称 + 序号）
2. 当前步骤高亮，已完成的步骤可点击回看，未到达的步骤灰色不可点
3. 聊天区底部出现「进入下一步 →」按钮（最后一步变为「完成」）
4. 点击进入下一步后，系统自动把上一步的对话内容整理后注入到新智能体的上下文中
5. 新智能体从第一句话起就能感知前面发生了什么

---

## 二、完整交互流程

```
首页工作流卡片
    ↓ 点击
/agents/智能体A?wf=工作流ID&step=0
    ↓ 用户与智能体A对话（普通聊天，无变化）
    ↓ 用户觉得这一步完成了，点「进入下一步」
/agents/智能体B?wf=工作流ID&step=1&from=会话A的ID
    ↓ 页面加载时自动读取会话A的消息，整理为上下文
    ↓ 用户与智能体B对话，B能看到A的对话摘要
    ↓ 点「进入下一步」（或「完成」）
/agents/智能体C?wf=工作流ID&step=2&from=会话B的ID
    ↓ 同上
    ↓ 最后一步点「完成」→ 回到首页
```

---

## 三、技术实现

### 3.1 首页工作流卡片改造

文件：`app/page.tsx`（工作流卡片区域）

- 原来：点击跳转到 `/agents/[agentCode]`
- 改为：拉取工作流的第一个步骤的 `agent_code`，跳转到 `/agents/[firstAgentCode]?wf=[wfId]&step=0`

### 3.2 聊天页新增进度条组件

文件：`app/agents/[id]/page.tsx`

新增逻辑：

1. **读取 URL 参数**：`wf`（工作流ID）、`step`（当前步骤序号，从0开始）、`from`（上一步会话ID）
2. **拉取工作流步骤**：调用 `/api/workflows/[wfId]/steps` 获取完整步骤列表（含每步的 agent_name、agent_code）
3. **渲染进度条**：步骤序号 + 智能体名称，当前步高亮，已完成步可点，未完成步灰色
4. **渲染「进入下一步」按钮**：固定在聊天底部右侧，最后一步显示「完成 ✓」
5. **步骤跳转**：点击「进入下一步」时
   - 取当前 `conversationId`（聊天过程中已赋值）
   - 拼接下一步 URL：`/agents/[nextAgentCode]?wf=[wfId]&step=[step+1]&from=[currentConvId]`
   - `router.push()` 跳转

### 3.3 上下文注入

文件：`app/agents/[id]/page.tsx` + `app/api/agents/[id]/chat/route.ts`

**前端**（page.tsx）：

- 页面加载时检测到 `from` 参数 → 调用 `/api/conversations/[fromConvId]/summary` 获取上一步摘要
- 若摘要存在直接用；若不存在，取最近 20 条消息拼成纯文本（格式：`用户：xxx\n助手：xxx`）
- 将整理好的文本存入组件 state：`workflowContext`

**首次发消息时**（page.tsx → chat API）：

- 请求体新增字段 `workflowContext: string`（仅第一条消息携带，之后置空）

**后端**（chat/route.ts）：

- 接收 `workflowContext` 字段
- 若存在，在构建 `messages` 数组时，在 agent 系统提示之后、历史消息之前，插入一条：
  ```json
  { "role": "system", "content": "【上一步工作记录】\n{workflowContext}" }
  ```
- 此条不入库，仅用于本次调用的上下文

### 3.4 步骤 API

文件（新增）：`app/api/workflows/[id]/steps/route.ts`

- GET：返回该工作流的所有步骤，含每步的 `step_order`、`agent_code`、`agent_name`（通过 join agents 表）
- 仅需用户已登录，无需管理员权限
- 供聊天页拉取步骤列表用

## 四、UI 设计

### 进度条样式（顶部）

```
─────────────────────────────────────────────────
  ① 需求分析 ✓  →  ② 方案设计（当前）  →  ③ 评审确认
─────────────────────────────────────────────────
```

- 已完成步：绿色圆圈 + 勾，文字可点击（跳回查看）
- 当前步：蓝色圆圈 + 加粗文字
- 未到达步：灰色圆圈，不可点

### 进入下一步按钮（底部右侧）

- 非最后一步：蓝色按钮「进入下一步 →」
- 最后一步：绿色按钮「完成工作流 ✓」→ 点击回到首页
