## 三、执行清单

### 1. 修复当前 lint 阻塞

当前 `npm run lint` 只卡一处：

- 文件：`app/admin/audit-logs/page.tsx`
- 位置：`useEffect` 内同步调用 `setLoading(true)`
- 规则：`react-hooks/set-state-in-effect`

建议处理方式：

- 保持页面行为不变。
- 只调整加载状态写法，避免 effect body 里直接同步 setState。
- 不扩大到审计日志筛选、分页、接口结构的重构。

验收：

```bash
npm run lint
```

必须通过。

### 2. 跑完整质量门禁

修完 lint 后执行：

```bash
npm run ci:check
```

重点看三项：

- `npm run ci:lint`
- `npm run ci:typecheck`
- `npm run ci:build`

备注：当前 `ci:test` 仍是占位 echo，不代表已有自动化测试覆盖。

### 3. 登录展示图手测

用 super_admin 在后台设置页验证：

- 上传 PNG / JPG / WEBP 成功。
- 后台预览立即更新。
- 刷新后台设置页后仍显示展示图。
- 登录页左侧展示图正常显示。
- 展示图上有 15% 黑色 overlay。
- 删除展示图后，后台预览与登录页都回退到默认深蓝纯色。
- 上传 SVG 被拒绝。
- 上传 GIF 被拒绝。
- 上传大于 5MB 的图片被后端拒绝。
- 上传大于 2MB 但小于 5MB 的图片有前端提示，并允许继续上传。
- org_admin / system_admin 直接调用上传或删除接口时返回 403。

涉及接口：

- `POST /api/admin/settings/login-showcase`
- `DELETE /api/admin/settings/login-showcase`
- `GET /api/settings`
- `GET /api/admin/settings`

### 4. 登录安全手测

验证登录错误文案收口：

- 不存在账号。
- 密码错误。
- 同一个手机号匹配多个账号。

以上三类都应返回同一类错误文案：账号或密码错误。

验证改密码踢旧会话：

- 用户自己改密码后，当前流程仍要求重新登录。
- 管理员改自己的密码后，当前会话不应立刻被自己踢掉。
- 管理员重置他人密码后，被重置用户的旧会话应失效，当前管理员会话不受影响。

### 5. 注册页同步决策

登录页当前已经进入双栏卡片结构，但注册页仍是旧版视觉。

收口后建议进入下一个小 patch：

- A：注册页共用登录页大背景和右侧卡片风格，左侧展示区复用 `login_showcase_url`。
- B：注册页只同步背景和表单卡片，不展示左侧图。
- C：注册页保持旧版，等后续统一登录 / 注册入口时再做。

倾向：A。

理由：用户从登录页点“立即注册”时，视觉连续性最好；也不需要新增 `register_showcase_url` 字段。

## 四、延期项

### 1. ENCRYPTION_KEY 拆分

当前 `lib/crypto.ts` 仍用 `JWT_SECRET` 派生 AES key。这个风险已在安全方案里记录，但不并入本次收口。

原因：

- 需要生产 DB 备份。
- 需要显式 old secret / new key 的迁移脚本。
- 需要 dry-run。
- 需要确认所有 `encrypt()` / `decrypt()` 调用点和所有加密字段。
- 做错会造成历史密文不可恢复。

下一步应作为独立 PR-B / 上线前专项执行。

### 2. migration_v34

待补一条 DB 迁移：

- 放宽 `logs.status` CHECK，允许 `'aborted'`。

要求：

- 新建 `supabase/migration_v34.sql`
- 更新 `supabase/MIGRATIONS.md`
- 不回改旧 migration。

### 3. trial 账号组织调整保护

`change_user_tenant` RPC 当前会把 trial 用户改成 personal / organization。已知问题暂不修，等体验版重新启用时补 trial 分支保护。
