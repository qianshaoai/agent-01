# 个人额度可视化 · 方案

版本：5.12up
分支：devB++
状态：设计稿，待评审

---

## 一、需求

用户能看到自己账号在不同时间窗口（**今天 / 本周 / 本月**）的调用次数，
以及自己**最常用的智能体**排行。

---

## 二、用户决策

| 项 | 选择 |
|---|---|
| 数据范围 | 自己（每个用户看自己） |
| 时间窗 | 自然月（每月 1 号清零，不是滚动 30 天） |
| 个人用户 vs 组织用户 | 共用入口；**个人用户隐藏进度条段**，只显示三档计数 + top3 |
| **进度条（组织用户）** | **画两根：上"公司额度"（`quota_used/quota`）+ 下"我累计贡献"（`logs 全期 success / quota`）** |
| **进度条 clamp** | "我累计贡献" 分子可能 > 分母（tenant 续过费/重置过），超过时 clamp 到 100% 并加文字 "已含历史 N 次" |
| **action 统计范围** | **只算 `action='chat'`**（与 tenant 扣费口径一致；workflow/upload/speech 不计） |
| **status 过滤** | **只算 `status='success'`**（aborted 不计、不扣费；error 也不计） |
| 配额耗尽 | 红色警告即可，详细收费提示后续补 |
| agent 名展示 | 直接读 `logs.agent_name` 快照字段，不 join agents |
| 入口 | 顶栏点击额度胶囊 → popover（阶段一）+ `/settings/usage` 详细页（阶段二） |
| 本期范围 | **只做阶段一 popover** |

## 五、后端 API

### 新建 `/api/me/usage` (GET)

**统一口径**：所有计数和聚合都加 `action='chat' AND status='success'` 过滤——和 `app/api/agents/[id]/chat/route.ts` 里"成功才扣 tenant.quota_used"的现行逻辑一致。aborted / error / workflow / upload / speech 都不计。

```typescript
// 5.12up · 当前用户的用量统计
{
  isPersonal: boolean,        // 个人用户标记，前端用来决定是否渲染进度条段
  quota: {
    orgUsed: number,          // tenants.quota_used（公司池）— 个人用户为 0
    orgTotal: number,         // tenants.quota — 个人用户为 0
    myTotal: number,          // 该用户全期 success+chat 累计（用于"我累计贡献"条）— 个人用户也填，但前端不画
    expiresAt: string | null, // tenants.expires_at — 个人用户为 null
  },
  counts: {
    today: number,            // 今天 0 点到现在
    thisWeek: number,         // 本周一 0 点到现在（中国习惯）
    thisMonth: number,        // 本月 1 号 0 点到现在
  },
  topAgents: Array<{
    agentCode: string,
    agentName: string,
    count: number,
  }>,                         // 本月范围 top 3，倒序
}
```

### 实现

`app/api/me/usage/route.ts`（新建）：

```ts
import { NextResponse } from "next/server";
import { db } from "@/lib/db";
import { getCurrentUser, requireFullUser } from "@/lib/auth";
import { apiError } from "@/lib/api-error";

const CHAT_FILTER = (q: any) =>
  q.eq("action", "chat").eq("status", "success");

export async function GET() {
  const user = await getCurrentUser();
  if (!user) return apiError("未登录", "UNAUTHORIZED");
  const guard = requireFullUser(user);
  if (guard) return guard;

  // 1. 公司配额（个人用户跳过）
  let orgUsed = 0, orgTotal = 0, expiresAt: string | null = null;
  if (!user.isPersonal && user.tenantCode) {
    const { data: t } = await db
      .from("tenants")
      .select("quota, quota_used, expires_at")
      .eq("code", user.tenantCode)
      .single();
    if (t) { orgUsed = t.quota_used; orgTotal = t.quota; expiresAt = t.expires_at; }
  }

  // 2. 时间窗（local 计算，toISOString() 自动转 UTC，logs.created_at 也是 UTC）
  const now = new Date();
  const today = new Date(now.getFullYear(), now.getMonth(), now.getDate());
  const dayOfWeek = (now.getDay() + 6) % 7; // 周一 = 0
  const monday = new Date(today); monday.setDate(today.getDate() - dayOfWeek);
  const monthStart = new Date(now.getFullYear(), now.getMonth(), 1);

  // 3. 四个 head-only count（今天/本周/本月/全期）+ 1 个本月明细（topAgents）
  //    全期 myTotal 用来画"我累计贡献"条
  const [todayRes, weekRes, monthRes, totalRes, monthLogsRes] = await Promise.all([
    CHAT_FILTER(db.from("logs").select("id", { count: "exact", head: true }))
      .eq("user_phone", user.phone).gte("created_at", today.toISOString()),
    CHAT_FILTER(db.from("logs").select("id", { count: "exact", head: true }))
      .eq("user_phone", user.phone).gte("created_at", monday.toISOString()),
    CHAT_FILTER(db.from("logs").select("id", { count: "exact", head: true }))
      .eq("user_phone", user.phone).gte("created_at", monthStart.toISOString()),
    CHAT_FILTER(db.from("logs").select("id", { count: "exact", head: true }))
      .eq("user_phone", user.phone),
    CHAT_FILTER(db.from("logs").select("agent_code, agent_name"))
      .eq("user_phone", user.phone).gte("created_at", monthStart.toISOString())
      .limit(2000),
  ]);

  // 4. 本月 top3 — JS 端 group by（Supabase JS 不支持 GROUP BY）
  //    单用户单月通常 < 500 条，2000 上限足够；超出场景见 §九
  const agentMap = new Map<string, { agentCode: string; agentName: string; count: number }>();
  for (const row of monthLogsRes.data ?? []) {
    const key = row.agent_code ?? "(unknown)";
    const cur = agentMap.get(key) ?? { agentCode: key, agentName: row.agent_name ?? key, count: 0 };
    cur.count += 1;
    agentMap.set(key, cur);
  }
  const topAgents = [...agentMap.values()].sort((a, b) => b.count - a.count).slice(0, 3);

  return NextResponse.json({
    isPersonal: user.isPersonal,
    quota: {
      orgUsed,
      orgTotal,
      myTotal: totalRes.count ?? 0,
      expiresAt,
    },
    counts: {
      today: todayRes.count ?? 0,
      thisWeek: weekRes.count ?? 0,
      thisMonth: monthRes.count ?? 0,
    },
    topAgents,
  });
}
```

**性能讨论**：
- 4 个 head-only count + 1 个本月明细，并发执行（`logs_user_phone_idx` 命中）
- 单用户全期 success 通常远低于 1k，count 在毫秒级
- topAgents JS group：< 500 条毫秒级
- 总耗时预估 100ms 内

**爆量预案**（万级 / 月）：改 RPC 函数做 SQL GROUP BY，本期不做。

## 六、前端 popover 组件

新建 `components/quota-popover.tsx`：

```typescript
"use client";
import { useEffect, useRef, useState } from "react";
import { AlertTriangle } from "lucide-react";

type Usage = {
  isPersonal: boolean;
  quota: { orgUsed: number; orgTotal: number; myTotal: number; expiresAt: string | null };
  counts: { today: number; thisWeek: number; thisMonth: number };
  topAgents: Array<{ agentCode: string; agentName: string; count: number }>;
};

export function QuotaPopover({ trigger }: { trigger: React.ReactNode }) {
  const [open, setOpen] = useState(false);
  const [data, setData] = useState<Usage | null>(null);
  const [loading, setLoading] = useState(false);
  const rootRef = useRef<HTMLDivElement>(null);
  const panelRef = useRef<HTMLDivElement>(null);

  // 打开时拉数据
  useEffect(() => {
    if (!open) return;
    setLoading(true);
    fetch("/api/me/usage")
      .then(r => r.ok ? r.json() : null)
      .then(d => setData(d))
      .finally(() => setLoading(false));
  }, [open]);

  // 点击外部关闭
  useEffect(() => {
    if (!open) return;
    function h(e: MouseEvent) {
      if (rootRef.current && !rootRef.current.contains(e.target as Node)) setOpen(false);
    }
    document.addEventListener("mousedown", h);
    return () => document.removeEventListener("mousedown", h);
  }, [open]);

  // Esc 关闭 + 焦点回到面板
  useEffect(() => {
    if (!open) return;
    function onKey(e: KeyboardEvent) {
      if (e.key === "Escape") setOpen(false);
    }
    document.addEventListener("keydown", onKey);
    panelRef.current?.focus();
    return () => document.removeEventListener("keydown", onKey);
  }, [open]);

  // 仅组织用户判断"耗尽"档（个人用户没有 tenant 池）
  const orgPct = data && !data.isPersonal && data.quota.orgTotal > 0
    ? Math.round((data.quota.orgUsed / data.quota.orgTotal) * 100)
    : 0;
  const exhausted = !!data && !data.isPersonal && orgPct >= 100;

  return (
    <div ref={rootRef} className="relative">
      <button
        onClick={() => setOpen(v => !v)}
        aria-haspopup="dialog"
        aria-expanded={open}
      >
        {trigger}
      </button>
      {open && (
        <div
          ref={panelRef}
          role="dialog"
          aria-modal="false"
          aria-label="我的用量"
          tabIndex={-1}
          className="absolute right-0 top-full mt-2 z-50 w-72 bg-white rounded-[14px] shadow-xl border border-gray-200 overflow-hidden focus:outline-none"
        >
          {exhausted && (
            <div className="bg-red-50 text-red-600 text-[12px] font-medium px-4 py-2 flex items-center gap-1.5 border-b border-red-100">
              <AlertTriangle size={13} /> 配额已用尽，请联系管理员
            </div>
          )}
          <div className="p-4">
            {loading ? <SkeletonView /> : data ? <ContentView data={data} orgPct={orgPct} /> : <ErrorView />}
          </div>
        </div>
      )}
    </div>
  );
}
```

子组件约定：

- **SkeletonView**：3 行灰色脉冲动画。
- **ContentView**：
  ```
  - 仅当 !data.isPersonal && data.quota.orgTotal > 0 时渲染"配额段"，包含两根条：
      • 公司额度：orgUsed / orgTotal，进度 = orgPct
        - orgPct ≥ 80 && < 100：进度条琥珀，"剩余 (orgTotal-orgUsed) 次" 红字
        - orgPct ≥ 100：进度条红色（顶部红条已显示）
      • 我累计贡献：myTotal / orgTotal，进度 = min(myTotal/orgTotal, 1) * 100
        - 灰色条
        - 若 myTotal > orgTotal：宽度 clamp 到 100%，文字补 "已含历史 (myTotal-orgTotal) 次"
  - 永远渲染"三档统计段"（今天/本周/本月）
  - 永远渲染"最常用 top 3"（topAgents 为空时显示 "本月暂无调用"）
  - 仅组织用户底部显示 "配额至 expiresAt"
  ```
- **ErrorView**：加载失败提示 + 重试按钮（重新触发 fetch）。

### 数据源划分（避免 trigger / popover 互相耦合）

- **trigger（顶栏胶囊）**：仍用 `app/page.tsx` 已有的 `quota` prop（`{ left, expiresAt }`，由 page 自身的 `getMe` 之类拉取），保持现状不动。

## 七点五、实施前确认事项

动手前必须先把这一项理清，否则查询口径会和生产实际不符：

### logs.status 实际允许值

- `supabase/schema.sql:110` 里 `CHECK (status IN ('success', 'error'))`，
- 但 `app/api/agents/[id]/chat/route.ts:317` 实际写入 `'aborted'`。

说明生产库有一条未入档的 migration（放松或删除了 CHECK 约束），或这条 insert 在某些环境上是静默失败的。

**在新 API 上线前**：
1. 在 Supabase SQL Editor 跑一次：
   ```sql
   SELECT status, count(*) FROM logs GROUP BY status;
   ```
   确认实际存在的值集。如果 `aborted` 真有写进去，方案当前的 `eq("status", "success")` 是对的；如果 `aborted` 一条都没有（说明 CHECK 在拦），那 abort 场景反而被算成 error 进 logs，需要回头看 chat route 的 abort 入库逻辑。
2. 若发现 schema.sql 与生产不一致，同步补 migration 文件入档（不在本方案范围内，作为遗留 TODO 记到变更记录里）。

### action 实际取值

同样跑一次：
```sql
SELECT action, count(*) FROM logs GROUP BY action;
```
确认除了 `chat / login / upload / speech` 外没有其他遗漏的、本该计入用量的 action。如果有 workflow 之类的，再回头讨论是否纳入。

## 九、风险 & 缓解

| 风险 | 缓解 |
|---|---|
| 个人用户没有 tenant 配额数据 | API 返回 `isPersonal: true` 且 `quota.orgTotal = 0`；ContentView 用 `!isPersonal && orgTotal > 0` 判断是否渲染整个进度条段 |
| logs 表有用户但 agent_name 为空（早期数据） | 显示 agent_code 作为兜底（API 已处理） |
| 自然周起点定义不一致（周日 vs 周一） | 决策：**周一开始**，符合中国习惯 |
| 单用户单月 > 2000 条调用 | 当前 limit 2000，溢出会丢部分明细影响 top3 准确性。出现这种用户量级时改成 RPC group by 处理；本期记 TODO |
| 跨页 popover 状态不一致 | 每次打开都重新拉数据，不缓存（数据本身变化频繁） |
| **"我累计贡献" 分子 > 分母**（tenant 续费/重置过 quota，但 logs 是永久的） | 进度条 clamp 到 100%，文字补 "已含历史 N 次"。不修正分子，因为没有 tenant 配额重置时间戳可用 |
| **进度条与 trigger 数据漂移**（两边都来自 `tenants.quota_used`，但分别拉取，时间窗内可能差 1～2 次） | 容忍。两边都是 tenant 共享池，用户感知不到 1-2 次差异；popover 打开瞬间会显示最新值 |
| **`status='aborted'` 不计入用量**（与 tenant 扣费口径一致） | 这是产品决策，文档已写明；FAQ 里如果有人问"为什么我点过的对话没算"，定位到这条 |
| **`action='chat'` 之外的调用不计入用量**（workflow / upload / speech） | 同上，与扣费口径一致；如果未来 workflow 也开始扣 tenant 配额，本 API 需要同步加上 |
