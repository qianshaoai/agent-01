## 2. 后端：新建 bulk API

`app/api/admin/users/bulk-set-tenant/route.ts`（新文件）：

- `POST /api/admin/users/bulk-set-tenant`
- Body: `{ userIds: string[], tenantCode: string | null }`
  - `tenantCode = null` 或 `'PERSONAL'` → 调到个人空间
  - 其它字符串 → 目标组织码
- 限制：
  - org_admin → 403
  - userIds 必填，非空，每项是字符串
  - 单次最多 200 人（`BULK_LIMIT`）
- 处理：
  - 一次拉所有目标用户的基础信息
  - 循环每人：跳过已删除 / super_admin / 已在目标组织内 → 收集到 failed[]
  - 其余调 RPC `change_user_tenant`，单条失败原因纳入 failed[]
  - 成功的写一条 audit_log（含 `detail.bulk=true` + `fromTenantCode`），`resourceTenantCode` 用目标组织（PERSONAL 则 null），便于 org_admin 在审计页看见迁入自己组织的事件
- 返回：`{ succeeded: [{id, phone}], failed: [{id, phone?, reason}] }`，整体永远 200

## 7. 用户列表角色优先级排序

用户反馈："让系统管理员排序优先级 > 组织管理员"。原 `.order("role")` 按字母序得到
`org_admin < super_admin < system_admin < user`，跟权限层级不符。

### migration_v33.sql

在 `users` 表加一个生成列 `role_priority`：

```sql
role_priority INT GENERATED ALWAYS AS (
  CASE role
    WHEN 'super_admin'  THEN 4
    WHEN 'system_admin' THEN 3
    WHEN 'org_admin'    THEN 2
    ELSE 1
  END
) STORED;
```

加索引 `idx_users_role_priority (role_priority DESC, created_at DESC)`。

### app/api/admin/users/route.ts

`.order("role")` 替换为 `.order("role_priority", { ascending: false })`。

效果：列表按 super → system → org → 普通用户 排，同优先级再按 created_at desc。

## 8. 用户列表筛选栏：级联三级筛选器

用户反馈："部门多了原来那个全部部门下拉会长到绕地球一圈"。决定把"全部组织 + 全部部门"
两个独立 select 合并为一个三级级联 dropdown，并取消"全部部门"独立列。

### 设计决策（用户拍板）

- 决策 1=A：点击节点名直接筛选 + 关闭下拉
- 决策 2=B：点击节点右侧 `>` 才展开下一级（不用 hover）
- 决策 3=A：按钮显示完整路径"组织 / 部门 / 小组"
- 决策 4：支持键盘 ↑↓ / → 展开 / ← 收起 / Enter 选择 / Esc 关闭
- 决策 5=B：触发按钮的箭头用下三角 `▼`（这是 dropdown，不是右展开）

### 后端：app/api/admin/users/route.ts

新增 `team_id` 查询参数（之前只支持 `dept_id` 和 `org`）。

### 前端：components/admin/scope-filter.tsx（新）

三级菜单组件 ScopeFilter：

- props：`tenants` / `depts` / `teams` / `value` / `onChange`
- 内部状态：`open` / `expandedTenant` / `expandedDept` / 三级高亮 index
- 触发按钮：显示路径或"全部组织"占位；右侧 ChevronDown，打开时旋转 180°；选中时显示 X 一键清空
- 菜单结构：
  - 一级 256px：列出所有组织，每行 [组织名|>]
  - 二级 176px：展开的组织下属部门，每行 [部门名|>]
  - 三级 144px：展开的部门下属小组，每行 [小组名]（无更下级）
  - 每级顶部都有"全部 X"项，选了就锁在该级
- 行内布局：左半 button 是 flex-1，右半 `>` button 固定 36px 宽 + 左边框 + hover 灰底，
  视觉上明显是两个可点的区域
- 溢出检测：打开时算 button 距右屏距离，不够 620px 时菜单整体右对齐 + flex-row-reverse
  避免被切走
- 键盘：所有箭头 + Enter + Esc
