# 对话导出方案

**文档日期**：2026-05-06
**适用范围**：正式版聊天页 `app/agents/[id]/page.tsx`
**文档目的**：让用户把当前与 AI 的对话导出成 Markdown 文件本地保存

---

## 一、需求

聊天页右上角加"导出"按钮，点击后下载当前 conversation 的全部消息为 `.md` 文件。

**只做 Markdown 一个格式**。其他格式（TXT / PDF / Word / JSON）等用户提了再加。

## 二、范围

### 允许改的文件

| 路径 | 改动 |
| --- | --- |
| `app/agents/[id]/page.tsx` | 加导出按钮 + 导出函数 |

### 不改的

- 后端任何接口
- 数据库
- 其他页面 / 组件
- 不引入新依赖（用 Blob + URL.createObjectURL 原生 API）

## 三、实现要点

### 1. 触发位置

聊天页顶部 header（深蓝渐变那条）右侧、quota 旁边加一个"导出"按钮。
- 当前 `activeConvId === null` 或 `messages.length === 0` 时**置灰**（没东西可导）
- 当前正在 `streaming` 时**置灰**（流式没结束）

### 2. 导出格式（Markdown）

```markdown
# 与 {agent.name} 的对话

**导出时间**：2026-05-06 15:30
**消息条数**：12

---

## 👤 用户  ·  14:21

你好

## 🤖 助手  ·  14:21

您好！请问有什么可以帮您的？

## 👤 用户  ·  14:23

请帮我列出表格内容

📎 附件：测试表格.xlsx

## 🤖 助手  ·  14:23

| 编号 | 名称 | 数量 |
| --- | --- | --- |
| 1 | 智能体A | 10 |
...

---

> 由「人机协同工作舱」生成 · {agent_code}
```

> 备注：原方案 footer 包含 `{agent.platform}`，但 `AgentInfo` 类型未带 platform 字段，
> `/api/agents/[id]` 接口也未返回。为了保持"对话导出方案不动后端"的边界，footer 收紧为只显示 agent_code。

### 3. 文件名

`{agent.name}-{conversation标题或时间}-{导出日期}.md`

例：`高净值理财智能体bot-你好-20260506.md`

文件名做合法字符过滤（去掉 `/\:*?"<>|`）。

### 4. 附件处理

- **图片附件**：导出 Markdown 用 `![filename](supabase_public_url)` 嵌入。文件不会失效（Supabase 公开 URL）。
- **文件附件**（PDF / xlsx / pptx 等）：在 user 消息块下方加一行 `📎 附件：filename.xlsx`（不嵌内容——内容已经在后端拼进 user content 里，会被 `parseUserContent` 剥离后单独显示）。
- **被中断的消息**：在末尾加 `⚠ 已停止生成`。

### 5. 时序

完全前端：
- 取 `messages` 数组（已经是渲染态，`parseUserContent` 处理过的干净文本 + `attachedFiles` / `attachedImages` 元数据）
- 拼字符串
- `new Blob([content], { type: "text/markdown;charset=utf-8" })` + `URL.createObjectURL` + 触发 `<a download>` 点击
- 不调任何 API，不动 DB，不扣配额

## 四、不做

- TXT / PDF / Word / JSON 多格式
- 后端导出接口（不需要）
- 批量导出多个聊天
- 导出到云盘 / 邮箱
- 自定义模板（导出格式固定）
- 导出按钮加 dropdown 选格式（先做单一 Markdown）

## 五、验收

| 编号 | 验收点 |
| --- | --- |
| E1 | 顶部 header 右侧出现"导出"按钮 |
| E2 | 空对话 / 流式中 按钮置灰不可点 |
| E3 | 点击后浏览器下载一个 `.md` 文件 |
| E4 | 文件名包含 agent 名 + 日期，特殊字符已过滤 |
| E5 | 内容用 `## 👤 用户` / `## 🤖 助手` 分块，每条消息上方有时间戳 |
| E6 | 图片附件以 `![](url)` 嵌入，能在 Markdown 渲染器里看到原图 |
| E7 | 文件附件以 `📎 附件：filename` 标注 |
| E8 | 被中断的消息末尾加 `⚠ 已停止生成` |
| E9 | 末尾加一行版权声明（"由...生成"）|
