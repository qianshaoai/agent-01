# 全应用字体大小切换方案

**文档日期**：2026-05-06
**适用范围**：用户端全应用（不含 admin 后台）
**文档目的**：让用户在「主页 → 设置」选择"大 / 中 / 小"三档字体，全应用文字跟着变；偏好仅记在本地浏览器，不跨设备同步

---

## 一、需求原文对齐

1. 字体大小作用范围：**全应用所有文字**（按用户原话）
2. 设置入口：**主页"设置"页面里**，点进去就能选大 / 中 / 小
3. 持久化：**localStorage**（这台机器/这个浏览器记住），不入 DB

---

## 二、范围

### 包含（受字体切换影响）

| 模块 | 路径 |
| --- | --- |
| 主页（工作流入口）| `app/page.tsx` |
| 智能体聊天页 | `app/agents/[id]/page.tsx` |
| 设置页 | `app/settings/**` |
| 登录 / 注册 | `app/login/**`、`app/register/**` |
| 用户自建智能体页 | `app/user-agents/[id]/**` |

### 不包含（保持原样）

| 路径 | 原因 |
| --- | --- |
| `app/admin/**` | 管理后台是另一拨人在用，密度高，调字体反而乱 |
| `app/trial/**` | 体验版独立模块，字号已专门调过 |

## 三、技术方案选型

### 现状

`grep` 统计：用户端 + 共享 components 里硬编码 `text-[Npx]` 共 **228 处**，分布：

| 字号 | 数量 |
| --- | --- |
| `text-[11px]` | 77 |
| `text-[12px]` | 51 |
| `text-[13px]` | 47 |
| `text-[10px]` | 20 |
| `text-[15px]` | 16 |
| `text-[14px]` | 9 |
| 其它 | 8 |

这些 px 是**绝对单位**，默认不会跟随 root font-size 变化。要让它们随用户偏好变化，要么改写、要么整体缩放、要么用 CSS 变量插值。

### 4 个候选方案

#### 方案 P1 · 改 230 处硬编码 + root font-size 切换

把所有 `text-[Npx]` 改成 tailwind 标准类（`text-xs / text-sm / text-base / text-lg / text-xl ...`）或 rem 形式（`text-[0.6875rem]`）；改 root font-size：小=14 / 中=16 / 大=18。

| 维度 | 评估 |
| --- | --- |
| 效果 | 全应用所有文字一起变，最贴合需求 |
| 工作量 | **1-2 天**（230 处搜索替换 + 全部页面视觉回归） |
| 风险 | 改动面大，可能引入意料外的视觉抖动；arbitrary px 改 tailwind 标准档时会有少量字号"跳档" |
| 副作用 | 按钮 / 徽章 / tag chip 文字也跟着变大，可能挤破 layout |

#### 方案 P2 · CSS `zoom` 整页缩放

`<html>` 加 inline style `zoom: 1.125`（大档）/ `1`（中档）/ `0.875`（小档）。**一行实现，所有字 + 图标 + 间距 + 图片等比缩放**。

| 维度 | 评估 |
| --- | --- |
| 效果 | 整页缩放（不只字体），按钮 / 图标 / 间距同步变大 |
| 工作量 | **30 分钟**（一个全局监听 localStorage 的 effect 改 html.style.zoom） |
| 风险 | `zoom` 不是 W3C 标准（但 Chrome / Edge / Safari / Firefox 117+ 都支持）；fixed 定位元素 / canvas / 第三方嵌入 iframe 可能位置错乱；浏览器自带 Ctrl+滚轮缩放跟它叠加可能怪 |
| 副作用 | 偏离用户字面"字体"概念——实际是"显示密度" |

#### 方案 P3 · 渐进式（root font-size + 仅改关键内容字号）

只改"内容性文字"（聊天气泡、卡片描述、表单 label、列表项标题）的 `text-[Npx]` 为 tailwind 标准类——大约 30-50 处；按钮 / 徽章 / 角标 / tag chip 等装饰性文字保持原样。

| 维度 | 评估 |
| --- | --- |
| 效果 | 主要内容字体跟着变；按钮等小装饰不变 |
| 工作量 | **半天**（识别 + 改 30-50 处） |
| 风险 | 视觉一致性：用户可能感到"只有部分文字变了"，与"全应用"的承诺有偏差 |
| 副作用 | 与字面需求有缝隙 |

#### 方案 P4 · CSS 变量 + 把硬编码 px 改成 inline style + calc

`--font-scale: 1`（小 0.875 / 中 1 / 大 1.125）；把 `text-[11px]` 改成 `style={{fontSize: 'calc(11px * var(--font-scale))'}}`。

| 维度 | 评估 |
| --- | --- |
| 效果 | 等比缩放，与 P1 类似但不用挑选 tailwind 档位 |
| 工作量 | 230 处全要加 inline style，**与 P1 同量级 1-2 天** |
| 风险 | inline style 多了不好维护；和现有 tailwind 风格风格冲突 |
| 副作用 | 代码可读性下降 |

## 四、推荐方案

**方案 P2 · zoom 整页缩放（试运行版，带 3 条强制约束）**。

理由：
1. **30 分钟交付**，今天就能上线，不打断本周节奏
2. 用户原话"全应用所有文字"在视觉上更接近"整页缩放"的体感（按钮、图标、间距等比放大）
3. 现有 `app/globals.css` (line 31-39) 已经有 `--fs-xs / --fs-sm / ... / --fs-3xl` 字号变量体系，但当前 228 处 `text-[Npx]` 大都没用这套体系。如果未来回退到 P1 风格，应该**复用 / 扩展**这套变量，避免再造轮子

如果 P2 验收发现某种 fixed / viewport 场景在 zoom 下破裂——优先改为**用户端容器级缩放**（在 `<main>` 而非 `<html>` 上 zoom）或回退到 P1（用 CSS 变量改 230 处）。**不再走"子元素 zoom: 1 白名单"路线，子元素 zoom 无法逃离祖先 zoom**。

## 五、实施细节（按 P2 推荐方案展开）

### 1. 设置入口

- 主页 `/settings` 页加一个"显示偏好"区块，里面放"字体大小"三选一（小 / 中 / 大）
- label 旁边小字注释：
  - "即时生效，仅本设备记住"
  - "会与浏览器自带的页面缩放（Ctrl+滚轮）叠加"
- 当前 `/settings` 页若没有合适分组，**新增"显示偏好"卡片到现有 settings 页**，不另开新路由

### 2. 持久化

- localStorage key：`app_font_size`
- 值：`"small" | "medium" | "large"`，默认 `"medium"`

### 3. 路由感知（必做）

**当前项目结构**：只有全局 `app/layout.tsx` 和全局 `components/providers.tsx`，**没有 admin / trial 专属 layout**。所以 zoom 不能"在 admin/trial 里强制设回 1"——从 admin 切回用户端时还要恢复，状态会互相踩。

**正确做法 · 路由级判断**：

- 新建 `lib/font-size.ts`：
  - `getFontSize()` 读 localStorage，返回 `"small" | "medium" | "large"`
  - `applyFontSize(pathname)`：
    - 路径**严格判断**：`pathname === '/admin' || pathname.startsWith('/admin/')`，trial 同理。避免 `/administrator` 这类未来路径被误判
  - 命中 admin / trial → `document.documentElement.style.zoom = ""`（清掉，回到默认）
    - 否则 → 按 localStorage 偏好设置 zoom 值
- 在 `components/providers.tsx` 中加一个 `<FontSizeBinder />` 组件，内部用 `usePathname()` 监听路由变化，每次变化时调用 `applyFontSize(pathname)`
- 设置页选项变化时立即写 localStorage + 调一次 applyFontSize

这样：
- 用户在 `/`、`/agents/...`、`/settings`、`/login`、`/register` → 应用偏好
- 用户切到 `/admin/dashboard` → zoom 立刻清掉
- 用户从 admin 切回 `/` → zoom 立刻应用偏好

### 4. zoom 档位

| 档位 | zoom 值 |
| --- | --- |
| 小 | `0.875` |
| 中 | `""` （清空 = 浏览器默认 1）|
| 大 | `1.15` |

不用 1.25 这种太大的值，避免在小屏幕上挤破 layout。

### 5. 首屏闪烁处理（必做）

localStorage 只能在客户端读，纯 React effect 方式会让大档/小档用户每次刷新先看到 100% 再跳到 115%/87.5%，**有可见跳动**。

**修法**：在 `app/layout.tsx` 的 `<html>` 加 `suppressHydrationWarning`（脚本会在 hydration 前给 `document.documentElement` 写 `style.zoom`，否则 React 会警告 attribute mismatch），并在 `<head>` 内插入 **pre-hydration inline script**：

```tsx
// app/layout.tsx
return (
  <html lang="zh-CN" suppressHydrationWarning>
    <head>
      {/* 必须放在 <body> 之前、其它脚本之前；hydration 前同步执行，零跳动 */}
      <script
        dangerouslySetInnerHTML={{
          __html: `
            try {
              var p = location.pathname;
              var isAdmin = p === '/admin' || p.indexOf('/admin/') === 0;
              var isTrial = p === '/trial' || p.indexOf('/trial/') === 0;
              if (!isAdmin && !isTrial) {
                var v = localStorage.getItem('app_font_size');
                if (v === 'small') document.documentElement.style.zoom = '0.875';
                else if (v === 'large') document.documentElement.style.zoom = '1.15';
              }
            } catch(e) {}
          `,
        }}
      />
    </head>
    <body>...</body>
  </html>
);
```

### 6. 与 globals.css 现有变量的关系

- 现有 `--fs-xs / --fs-sm / --fs-base / --fs-md / --fs-lg / --fs-xl / --fs-2xl / --fs-3xl` 体系不动
- 现有 root font-size: 17px 不动
- zoom 是**层级无关**的整体渲染缩放，跟字号变量、Tailwind 标准档、px 硬编码全部相容
- 未来如果回退到 P1 / P3 路线，应该**直接用这套 fs-* 变量替换 230 处硬编码**，而不是另立 tailwind 标准档

## 八、风险与回退预案

### 已知风险

1. **某些浏览器 / 极端 zoom 值下 layout 破裂**：限制档位最大 1.15（不超过 +15%），降低风险面。
2. **zoom + Ctrl+滚轮 双重缩放**：浏览器自带的页面缩放跟 CSS zoom 是叠加的。设置项旁加注释提醒"会与浏览器自带缩放叠加"。
3. **未来移植到 webview / 桌面壳**：某些 webview 对 zoom 支持差。这种场景目前不存在。
4. **首屏闪动**：通过 `<head>` 里 pre-hydration script 解决；如果某种边缘情况这段脚本失败（极少见），最差表现是闪一次后 hydration effect 兜底设置正确，不会卡住。

### 回退路径（评审修正版）

**不再使用"子元素 zoom: 1 白名单"思路**——子元素 zoom 无法逃离祖先 zoom 的渲染缩放，这条路死路。

如果上线后某种 fixed / viewport 场景出现难以接受的副作用：

- **路径 A · 容器级缩放**：把 zoom 应用从 `<html>` 改到用户端的 `<main>` 或某个统一容器（admin / trial 不在这一层之下）。**改动量中等**——当前项目没有用户端统一容器，真改可能要给主页、聊天页、设置页、登录注册、自建智能体各自包一层；且 fixed / toast 不一定在这个容器内，需要单独处理。
- **路径 B · 退到 P1（CSS 变量）**：复用现有 `app/globals.css` 已定义的 `--fs-xs / --fs-sm / --fs-base ...` 变量，把 230 处 `text-[Npx]` 改成 tailwind 标准类或对应变量。工作量 1-2 天。这条是"彻底解"。
- **路径 C · 快速止血**：改 default 为 `"medium"` 并清空 localStorage。本方案核心代码集中在 `lib/font-size.ts` + Providers 一个 binder + 设置页一个区块，**全删不超过 100 行**。
