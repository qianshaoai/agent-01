## 二、tenantName bug 修复（小改）

**问题**：登录后顶部 badge "CESHI · CESHI"，蓝白两半都显示组织编码而不是组织名称。

**根因**：`app/api/auth/login/route.ts` 之前用 `user.tenant_code` 作为 tenantName 兜底，没去 tenants 表读 name。

**改动**：
- `app/api/auth/login/route.ts` — 登录时把 `tenants.name` 写进 JWT 的 tenantName 字段
- `app/api/me/route.ts` — `/api/me` 返回时优先用 DB 的最新 name，覆盖 JWT 里旧的 tenantName（让老 cookie 用户不用重登）

**回滚**：
- 还原两个文件到 `e886538` 之前版本即可

**为什么没单独写方案**：纯 bug 修复，3 行改动，实施前已征得用户认可。

## 三、上传支持 .md 文件（小改）

**问题**：上传 markdown 文件被前端拒绝。

**根因**：`app/api/upload/route.ts` 的 `ALLOWED_TYPES` 只用 MIME 查表，markdown 浏览器报的 MIME 是空 / `text/markdown`，都不在表里。

**改动**：
- `app/api/upload/route.ts`
  - 加 `text/markdown` / `text/x-markdown` 到 `ALLOWED_TYPES`
  - 加 `EXT_TO_TYPE` 兜底（MIME 查不到时按扩展名匹配，覆盖浏览器空 MIME 的情况）
  - magic bytes 校验：md 跟 txt/csv 一样跳过
  - 提取分支：md 走 `decodeText`

**回滚**：
- 还原 `app/api/upload/route.ts` 到原版本

**为什么没单独写方案**：纯能力扩展（之前漏了 .md），实施前已征得用户认可。

## 四、JWT_SECRET 同步（环境配置）

**问题**：本地 dev 的 `JWT_SECRET` 跟线上不一致，导致本地无法解密线上 admin 后台创建的 agent 的 api_key_enc，调 Coze 时 token 错误。

**改动**：
- `.env.local` 的 `JWT_SECRET` 同步成线上 Jenkins 凭据 `agent01_JWT_SECRET` 的值

**注意**：`.env.local` 在 .gitignore 里，**不影响 git / 任何分支**。这条仅记录用户开发环境的一次性同步。
