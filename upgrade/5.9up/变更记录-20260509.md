一、主菜：5.9up 历史工作流

需求：用户能查看之前用过的工作流（已完成 / 已放弃），只读回看每一步对话。

设计文档：[`upgrade/5.9up/历史工作流-方案-20260509.md`](./历史工作流-方案-20260509.md)

1.1 数据库

不需要 migration。`workflow_sessions.status` 已支持 `in_progress / completed / abandoned` 三态；`conversations.session_id` 已建索引。

1.2 后端 API 改动

| 路由 | 改动 |
| --- | --- |
| `app/api/workflow-sessions/[id]/route.ts` | DELETE 默认行为改成软删除（`UPDATE status='abandoned'`）；`?hard=1` 走物理删除，并校验当前 status 不是 `in_progress` |
| `app/api/workflow-sessions/route.ts` | GET 接受 `?status=history`，转成 `IN ('completed', 'abandoned')` 过滤 |

1.3 前端新页面

| 文件 | 功能 |
| --- | --- |
| `app/workflows/history/page.tsx` | 列表页：搜索框、状态筛选（全部/已完成/已放弃）、行操作（查看 / 永久删除），按 updated_at 倒序 |
| `app/workflows/history/[id]/page.tsx` | 详情页：会话信息 + 每步徽章 + "回看对话"链接（只读模式入口） |

1.4 智能体页 readonly 模式

URL `?readonly=1&conv=<id>`：

- 顶部黄色 banner "只读模式 · 历史会话回看"
- 整块输入区不渲染（包括上传 / 语音 / 发送按钮）
- 消息上的「编辑 / 重新生成」隐藏，"复制"保留
- 「下一步」按钮不渲染
- 进度条 outline 自动发送被跳过
- 返回箭头指向 `/workflows/history/[sessionId]`
- 侧边栏「新建对话」「重命名 / 删除」全部隐藏（避免误触）

1.5 中间件

`middleware.ts` 把 `/workflows` 加入 USER_PROTECTED 列表，未登录跳 `/login`。

二、主菜：组织管理员可见权限可选

需求：org_admin 应能在本组织范围内自选可见性（全员 / 部门 / 小组），而非 5.7up 的强制"全组织"。

设计文档：[`upgrade/5.9up/组织管理员可见权限-方案-20260509.md`](./组织管理员可见权限-方案-20260509.md)

2.1 后端

`app/api/admin/workflows/route.ts` 和 `[id]/route.ts` 新增 `validateOrgAdminPermissions()` 工具：校验 org_admin 提交的 permissions 中所有 dept_id / team_id 必须真实属于 admin 的 tenantCode；scope=org 必须等于本组织 code。

- POST：org_admin 不传 permissions → 默认全组织（`org_only` + scope=org）；传了 → 校验后保存
- PATCH：放开 visibleTo 写权限（限定 `org_only` / `custom`）；放开 permissions 写权限（同样校验）；空 permissions + `org_only` 时自动补 scope=org（保存"全员可见"不变更时不会被空数组校验拒绝）

2.2 前端

`app/admin/workflows/page.tsx`：

- `useEffect` 在 `/api/admin/me` 拉取时同时读 `tenantCode`，存入 `adminTenantCode`
- 把蓝色不可修改提示替换成 select：org_admin 模式下三选项（"我所在组织全员可见 / 指定本组织部门可见 / 指定本组织小组可见"）
- dept/team picker 在 org_admin 模式下用 `tenants.filter(t => t.code === adminTenantCode)` 过滤，只展示本组织的部门 / 小组
- `openEditWf` 智能映射：当 `visible_to='org_only'` 且带 scope=org permission 时（5.9up 新语义），非 org_admin 视角下转成 `custom:org` 展示，避免下拉显示成误导性的 "仅组织用户可见"

2.3 Badge 显示优化

工作流列表行的可见性 badge：

- `org_only` 无 scope=org（旧"任意组织用户"语义）→ "仅组织用户"（蓝）
- `org_only` 带 scope=org（新语义，限定组织）→ "指定组织：xxx"（橙）
- `custom` 模式：把 scope_id 解析为名称
  - 部门：`前哨测试公司 / 测试部门D`（带母公司前缀，跨组织好辨认）
  - 小组：`前哨测试公司 / 测试部门D / A组`（三级路径）
- 多个时全部铺出来（不是 `指定 X（N个）`），用顿号分隔

四、Bug 修复

| 问题 | 原因 | 修复 |
| --- | --- | --- |
| 「上一步」按钮丢 sessionId，之后所有对话能跨 session 看到历史 | URL 没拼 `&session=` | 拼上 sessionId 参数 |
| 主加载 useEffect 不响应 sessionId 变化 | deps 只有 `[agentCode]` | 加上 `sessionId` / `readonly` / `initialConvId` |
| 跨步骤上下文 useEffect 同样问题 | deps 缺 sessionId | 加上 sessionId；ref 在变化时清空 |
| 发完消息后侧边栏拉回所有跨 session 对话 | 流结束后的 conversation 列表刷新 URL 漏带 sessionId | 拼上 sessionId 参数 |
| 「继续」回到旧步骤而不是当前步骤 | advanceFrom 的 PATCH 用 `fetch().catch()` 没等待，紧随 `window.location.assign` 把请求取消，DB 里 currentStepIdx 没更新 | 三处 PATCH 加 `keepalive: true` |
| 历史详情页步骤列表为空 | 解析 API 返回当数组（实际是 `{ steps: [...] }` 对象）| 改成读 `data.steps` |
| 「上一步」自动发"请回顾对话"废话 | URL 带了 `outline=1` | 删掉 outline 参数；进入页面自动加载该 agent 在本 session 的现有对话（恢复历史窗口而不是空白） |
| 工作流详情页步骤按钮丢 session | `WorkflowStepButton` 不接收 sessionId prop | 加 `sessionId` prop；首页 `WorkflowStepButton` 根据 `mySessions` 里该工作流的进行中会话自动透传 |
| Modal autoFocus 触发页面滚动 | autoFocus 在 fixed 弹窗里浏览器 scroll-into-view 计算错乱 | 改用 `useRef` + `focus({ preventScroll: true })` |
| Lint warning：useEffect 缺依赖、unused `fromConvId` | 5.8up 残留代码 | 加齐 deps；删 unused var |
