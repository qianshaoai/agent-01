## 1. 本阶段目标

只解决一件事：**节点内快捷绑定智能体**。

1. 未绑定 agent 的 `agent` 类节点显示"绑定智能体"按钮
2. 点击按钮 → 轻量浮层（搜索 + 列表）→ 选中即保存
3. 节点内补充展示绑定智能体类型徽章（chat / external）
4. 智能体已删除时仍能通过同一浮层重新绑定
5. 整个工作流展开期间智能体列表只请求 1 次

## 5. 实现要点（不构成代码强约束）

### 5.1 浮层定位
- 锚点：节点内"绑定智能体"按钮
- 位置：节点正下方或右侧弹出，避免覆盖相邻节点
- 关闭条件：点选 / 按 Esc / 点击浮层外区域

### 5.2 浮层内容
```
┌─────────────────────────┐
│ 🔍 搜索智能体...         │
├─────────────────────────┤
│ 🤖 案例撰写专家  [chat]  │ ← 点击即绑定
│ 🤖 数据分析助手  [chat]  │
│ 🔗 外部审核工具  [外链]  │
│ ...                      │
└─────────────────────────┘
```

### 5.3 数据缓存策略
- agents 列表在工作流展开时由父组件加载一次
- 浮层通过 props 接收 agents 数组
- 同一工作流多个未绑定节点点击浮层共享数据，**不重复请求**

### 5.4 PATCH 请求（接口适配，必读）

**现有接口 `app/api/admin/workflow-steps/[id]` 的关键逻辑**：

```ts
if (body.agentId !== undefined) {
  updates.agent_id = updates.exec_type === "agent" ? (body.agentId || null) : null;
}
```

也就是说，**如果只传 `agentId` 不传 `execType`，`updates.exec_type` 是 `undefined`，最终会把 `agent_id` 写成 `null`，绑定失败**。

**正确请求体**（必须严格遵守）：

```json
{ "execType": "agent", "agentId": "xxx" }
```

**约束**：
- 阶段二**不允许**改后端 `app/api/admin/workflow-steps/[id]/route.ts`，必须靠正确的请求体绕过现有逻辑
- 乐观更新：先在前端把节点 `agent_id` 字段更新，PATCH 失败则回滚
- 用现有 toast / 错误处理机制提示

### 5.5 agents 列表数据上限

- `/api/admin/agents` 默认 `pageSize=50`，上限 100
- 浮层加载时显式传 `?pageSize=100`
- 超过 100 个智能体的场景**不在阶段二范围**，作为已知短板独立立项，未来需要专门的轻量候选接口（带搜索、不分页）

## 7. 风险点

| 风险 | 缓解 |
|------|------|
| 浮层与节点横向滚动冲突 | 浮层用 `position: absolute` 跟随节点；滚动时随节点移动或自动关闭 |
| agents 列表很大导致浮层卡 | 浮层内列表限定最高 320px + 内部滚动；搜索过滤前端做 |
| PATCH 乐观更新与失败回滚状态错位 | 失败回滚到原 `agent_id`，弹错误 toast |
| 多人同时编辑产生覆盖 | 阶段二不解决并发，沿用现有"最后写入获胜"语义 |
| 阶段一组件被修改幅度过大 | 只允许在节点渲染内插入分支，不允许重构节点结构 |
