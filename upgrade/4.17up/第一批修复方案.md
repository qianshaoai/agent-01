## F1：拆出 `NavContent` 组件

### 问题位置
`components/layout/admin-layout.tsx` 第 120-191 行

### 问题描述
`NavContent` 是一个组件（以 `<NavContent />` 方式使用），但定义在父组件函数内部。每次父组件 re-render 都会重新创建这个函数引用，导致 React 把它当成一个**新组件**——子组件状态全部丢失，所有子孙元素重新挂载。

### 报错信息
```
react-hooks/static-components:
Cannot create components during render.
Components created during render will reset their state each time they are created.
Declare components outside of render.
```

### 修改方案

**方案 A（推荐）：抽到组件外部，显式传参**

将 `NavContent` 定义移到 `AdminLayout` 函数外部，把它需要的所有 state/值通过 props 传入：

```tsx
// 文件顶部，AdminLayout 函数之前
type NavContentProps = {
  pathname: string;
  siteSettings: { logo_url: string; platform_name: string };
  adminUsername: string;
  adminRole: AdminRole;
  visibleNavGroups: typeof navGroups;
  onItemClick: () => void;
};

function NavContent({
  pathname,
  siteSettings,
  adminUsername,
  adminRole,
  visibleNavGroups,
  onItemClick,
}: NavContentProps) {
  return (
    <>
      {/* 原来 120-190 行的 JSX，将 setMobileOpen(false) 替换为 onItemClick() */}
    </>
  );
}

// AdminLayout 函数内部
<NavContent
  pathname={pathname}
  siteSettings={siteSettings}
  adminUsername={adminUsername}
  adminRole={adminRole}
  visibleNavGroups={visibleNavGroups}
  onItemClick={() => setMobileOpen(false)}
/>
```

**方案 B（次选）：改为渲染函数**

不定义为组件，改为普通函数返回 JSX 节点：

```tsx
const navContent = (
  <>
    {/* 原来的 JSX */}
  </>
);

// 使用时
<aside>{navContent}</aside>
```

这样不是组件而是 JSX 节点，React 不会把它当成新组件。

**推荐方案 A**：代码更清晰，props 显式，后续如需加 memo 也方便。

## F2：`admin-layout.tsx` localStorage 初始化

### 问题位置
`components/layout/admin-layout.tsx` 第 77、81-85 行

### 问题描述
```tsx
const [siteSettings, setSiteSettings] = useState({ logo_url: "", platform_name: "" });

useEffect(() => {
  try {
    const cached = localStorage.getItem("brand_settings_v1");
    if (cached) setSiteSettings(JSON.parse(cached));  // ← set-state-in-effect
  } catch {}
  // ...
}, [pathname]);
```

`useEffect` 同步调用 `setSiteSettings`，触发 React 19 的 `set-state-in-effect` 规则。

### 修改方案

将 localStorage 读取挪到 `useState` 的初始化函数中，避免在 effect 内同步 setState：

```tsx
const [siteSettings, setSiteSettings] = useState(() => {
  if (typeof window === "undefined") return { logo_url: "", platform_name: "" };
  try {
    const cached = localStorage.getItem("brand_settings_v1");
    if (cached) return JSON.parse(cached);
  } catch {}
  return { logo_url: "", platform_name: "" };
});

useEffect(() => {
  // 异步请求保留在 effect 中
  fetch("/api/settings")
    .then((r) => r.json())
    .then((d) => {
      setSiteSettings(d);
      try { localStorage.setItem("brand_settings_v1", JSON.stringify(d)); } catch {}
    })
    .catch(() => {});
  // refreshMe 相关逻辑保留
}, [pathname]);
```

**要点**：
- `useState(() => init)` 惰性初始化是允许的
- 注意 SSR 兼容：`typeof window === "undefined"` 时返回默认值，避免 hydration 不一致

## F3：`analytics/page.tsx` useEffect 自动加载重构

### 问题位置
`app/admin/analytics/page.tsx` 第 60 行

### 问题描述
```tsx
const load = useCallback(async () => {
  setLoading(true);
  // ... fetch，然后 setData、setLoading
}, [days, tenantFilter, deptFilter, teamFilter, userSearch]);

useEffect(() => { load(); }, [load]);  // ← 报错
```

`useEffect` 内直接调用 `load()`，而 `load()` 内部同步更新 state，触发规则。

### 修改方案

**方案 A（推荐）：把请求逻辑直接写进 useEffect**

把 `load` 拆成"纯请求参数"和"执行"两部分，useEffect 只负责执行：

```tsx
useEffect(() => {
  let cancelled = false;
  const params = new URLSearchParams({ days: String(days) });
  if (tenantFilter) params.set("tenantCode", tenantFilter);
  if (deptFilter) params.set("deptId", deptFilter);
  if (teamFilter) params.set("teamId", teamFilter);
  if (userSearch) params.set("userSearch", userSearch);

  setLoading(true);
  fetch(`/api/admin/analytics?${params}`)
    .then((r) => r.ok ? r.json() : null)
    .then((d) => {
      if (cancelled) return;
      if (d) setData(d);
      setLoading(false);
    })
    .catch(() => { if (!cancelled) setLoading(false); });

  return () => { cancelled = true; };
}, [days, tenantFilter, deptFilter, teamFilter, userSearch]);
```

同时删除原来的 `load` useCallback（如果页面其他地方还用，改为事件回调而非 useCallback 形式）。

**要点**：
- 增加 `cancelled` 标志处理快速切换筛选时的竞态
- useEffect 直接执行 fetch，不经过中间函数包装，规则不再报错
- 状态更新发生在 Promise 回调中，不是 effect 同步阶段

## F4：`logs/page.tsx` useEffect 重构

### 问题位置
`app/admin/logs/page.tsx` 第 54-55 行

### 问题描述
```tsx
useEffect(() => { setPage(1); load(1); }, [statusFilter, tenantFilter, pageSize]);
useEffect(() => { load(page); }, [page]);
```

两个 useEffect 都在 effect 中直接 setState + 发请求，且两者相互影响（第一个 setPage(1) 会触发第二个），容易重复请求。

### 修改方案

合并为单个 useEffect，并用 cancelled 标志防竞态：

```tsx
// 筛选条件变化时只重置页码，不再直接调 load
useEffect(() => {
  setPage(1);
}, [statusFilter, tenantFilter, pageSize]);

// 统一在这里监听 page 和所有筛选条件变化，发起请求
useEffect(() => {
  let cancelled = false;
  const params = new URLSearchParams({ page: String(page), pageSize: String(pageSize) });
  if (search) params.set("search", search);
  if (statusFilter !== "all") params.set("status", statusFilter);
  if (tenantFilter !== "all") params.set("tenantCode", tenantFilter);

  setLoading(true);
  Promise.all([
    fetch(`/api/admin/logs?${params}`).then((r) => r.json()),
    fetch("/api/admin/tenants").then((r) => r.json()).then((d) => d.data ?? d),
  ]).then(([lr, tr]) => {
    if (cancelled) return;
    setLogs(lr.data ?? []);
    setTotal(lr.pagination?.total ?? 0);
    setTenants(Array.isArray(tr) ? tr : []);
    setLoading(false);
  }).catch(() => { if (!cancelled) setLoading(false); });

  return () => { cancelled = true; };
}, [page, pageSize, statusFilter, tenantFilter]);

function handleSearch(e: React.FormEvent) {
  e.preventDefault();
  setPage(1);
  // 若 page 本来就是 1，effect 不会重跑，需要主动触发一次
  // 可以给 load 独立保留一个版本，或用一个 "refreshTick" state 手动触发
}
```

**要点**：
- 把"筛选变化 → 重置 page"和"page 变化 → 发请求"拆成两步单向依赖
- `handleSearch` 需要注意：如果当前已经在 page 1，`setPage(1)` 不会触发 effect，需要额外引入一个 `refreshTick` state 来强制刷新：

```tsx
const [refreshTick, setRefreshTick] = useState(0);

// effect 依赖加上 refreshTick
}, [page, pageSize, statusFilter, tenantFilter, refreshTick]);

function handleSearch(e: React.FormEvent) {
  e.preventDefault();
  if (page === 1) setRefreshTick(n => n + 1);
  else setPage(1);
}
```

## F7：`doc/read_doc.cjs` 处理

### 问题位置
`doc/read_doc.cjs` 第 1 行：`const mammoth = require('../node_modules/mammoth');`

### 报错信息
```
@typescript-eslint/no-require-imports: A `require()` style import is forbidden
```

### 修改方案

这是个临时文档读取脚本，不属于主工程代码。推荐：

**方案 A（推荐）：从 lint 范围中排除 `doc/` 目录**

修改 `eslint.config.mjs`，在 ignores 中加入 `doc/**`：

```js
export default [
  // ... 原有配置
  {
    ignores: ["doc/**", "node_modules/**", ".next/**", ...],
  },
];
```

**方案 B：改为 ESM**

把 `.cjs` 改名为 `.mjs` 并用 import 语法：

```js
// read_doc.mjs
import mammoth from "mammoth";
const r = await mammoth.extractRawText({ path: "..." });
console.log(r.value);
```

注意已经存在 `doc/read_doc.mjs`（运行检查方案中提到），所以可以直接**删除 `read_doc.cjs`**，只保留 mjs 版本。

**推荐方案 A**：`doc/` 目录本身就是文档存放地，不应该被 lint 卡住主工程质量门禁。
