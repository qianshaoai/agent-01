## 一、聊天页 UX 微调

### 1. Markdown fenced 代码块加复制按钮

**问题**：bot 输出 ```` ```text ... ``` ```` 代码块时，要复制内容只能整段拖蓝复制，会带上前后描述文字。

**实施**：
- `app/agents/[id]/page.tsx` 新增 `CopyableCodeBlock` 组件
- ReactMarkdown `components.pre` 接管 `<pre>` 渲染：包一层 `<div className="relative group/code">`，右上角放复制按钮
- 按钮点击：`navigator.clipboard.writeText(pre.textContent)`，只取代码块内容
- 默认隐藏，hover 代码块时浮现；已复制 1.5s 内显示反馈状态后还原

**视觉迭代**：
- 第一版：黑底"复制"文字标签 → 用户反馈太丑
- 第二版（当前）：28×28 透明方块 + lucide `Copy` 图标 / 复制后变 `Check` 绿色 + hover 浅灰背景

### 2. 流式生成期间不再强制滚到底

**问题**：bot 流式输出时，每收一段 delta 都把页面滚到底部，用户想边生成边往上看历史不可能。

**实施**：
- `app/agents/[id]/page.tsx` 把自动滚动 useEffect 的依赖从 `[messages]` 改成 `[messages.length]`
- 流式中只是末尾 assistant 的 `content` 变长（同一条消息），messages.length 不变 → 不触发滚动
- 用户发送 / 切换会话 / 重新生成时 length 变化 → 仍会滚到底（保留）
- 想跟最新内容右下角"↓ 回到最新"按钮已存在，无需额外做

## 二、字体大小切换重做（路径 A 已实施）

**问题**：旧方案 `zoom: 1 / 1.15 / 1.30` 是页面密度缩放，不是字体缩放——文字变大的同时 padding / 间距 / 卡片 / 按钮也跟着放大，导致中 / 大档界面变挤、固定定位元素被推出可视区。

**新方案（A 路径，已落地）**：

只放大字号，不放大布局。

- `lib/font-size.ts`：删 `ZOOM_MAP`，`applyFontSize(level)` 改为给 `<html>` 写 `data-font-size="small|medium|large"`，并清理旧 `style.zoom`
- `app/globals.css`：加 root font-size 规则
  - 默认（small） — 不写规则，沿用浏览器默认 16px
  - `:root[data-font-size="medium"] { font-size: 19px; }`
  - `:root[data-font-size="large"]  { font-size: 21px; }`
- `app/layout.tsx`：pre-hydration script 从设置 `style.zoom` 改为设置 `data-font-size` 属性，避免 SSR 与首屏闪烁
- Tailwind v4 的 `text-xs/sm/base/lg` 等是 rem 单位 → 自动跟着 root font-size 缩放
- 项目里大量 `text-[NNpx]` 任意值类**不会**跟着放大，覆盖率 ~50-70%（路径 B 暂未做，等观察效果）

## 四、5.7up 主菜：管理员权限完善（三层管理员）

需求来源：桌面 `管理员功能完善.docx`，方案落在 `upgrade/5.7up/管理员权限完善-方案-20260507.md`。

**三层角色**：
- `super_admin` — 平台拥有者
- `system_admin` — 平台运维
- `org_admin` — 组织管理员（绑定 `tenantCode`，仅看本组织）

**5 项决策（用户拍板）**：
1. org_admin 看 workflows，按 `resource_permissions` 范围（org/dept/team）过滤
2. org_admin 创建 workflow 时强制 `visible_to='org_only'`，并自动写一条 scope=org 的权限
3. org_admin 可以编辑本组织内**所有** workflow（不仅自己建的）
4. org_admin 对 `/api/admin/agents` 仅 GET，所有写操作返回 403
5. super_admin 走前台登录时拒绝："请登录后台"

### 4.1 后端 API 权限边界

| 路由 | 改动 |
| --- | --- |
| `/api/auth/login` | super_admin 走前台直接 403 "请登录后台" |
| `/api/admin/tenants` `GET` | org_admin 限定 `query.eq("code", admin.tenantCode)` |
| `/api/admin/tenants` `POST` | org_admin → 403 |
| `/api/admin/tenants/[id]` `PATCH` `DELETE` | org_admin → 403 |
| `/api/admin/departments` `GET` `POST` | org_admin 强制 `tenant_code = admin.tenantCode` |
| `/api/admin/departments/[id]` `PATCH` `DELETE` | org_admin 通过 `ensureOrgScope` 校验所属 |
| `/api/admin/teams` `GET` `POST` | 同上 |
| `/api/admin/teams/[id]` `PATCH` `DELETE` | 同上 |
| `/api/admin/workflows` `GET` | org_admin 走 `resource_permissions` 取 scope=org/dept/team 集合，再 `in("id", scopedWfIds)` |
| `/api/admin/workflows` `POST` | org_admin 强制 `visible_to='org_only'`，自动写 org 范围权限行 |
| `/api/admin/workflows/[id]` `PATCH` `DELETE` | 新增 `ensureOrgAdminCanTouch`：必须有本 admin tenantCode 内的 scope=org/dept/team 权限行；org_admin 不可改 `visibleTo`、不可重写权限 |
| `/api/admin/agents` `GET` | 允许 org_admin |
| `/api/admin/agents` `POST` 与 `[id]/(PATCH|DELETE)` | org_admin → 403 |
| `/api/admin/settings/route.ts` `/logo/route.ts` `/contact-qr/route.ts` | 三个写接口仅 super_admin |
| `/api/workflows` (用户端) | system_admin 直通；org_admin 仍按权限范围过滤 |

辅助：`ensureOrgScope(scope_type, scope_id, tenantCode?: string | null)` 收容空判 + 跨组织拒绝。
