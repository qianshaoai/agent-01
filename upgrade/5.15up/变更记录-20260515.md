## 三、dev 稳定性补丁（开发期跨境 Supabase 抖动）

### 3.1 lib/db.ts 加 retry wrapper（根治）

```ts
async function retryingFetch(input, init) {
  for (let i = 0; i < 3; i++) {
    try { return await fetch(input, init); }
    catch (e) { if (i === 2) throw e; await new Promise(r => setTimeout(r, 200 * (i+1))); }
  }
}
const db = createClient(..., { global: { fetch: retryingFetch } });
```

所有 `db.from(...)` 内部自动 retry 3 次 / 200ms / 400ms 退避，只对 fetch throw（ECONNRESET / TLS handshake 失败）触发，HTTP 4xx/5xx 不重试。

### 3.2 前端列表 retry 3 次（缓解 dev 严格模式双调）

`/admin/model-providers/page.tsx` + `/admin/agent-builder/page.tsx` 的 `loadList` 改 3 次重试，全失败才 flash err，避免 React Dev 严格模式 useEffect 双调时 mount-1 失败 mount-2 成功导致红条残留。

### 3.3 修一类隐藏 bug：fetch failed 被当 NOT_FOUND

| 路由 | 现象 | 修法 |
|---|---|---|
| `/api/admin/agent-drafts/[id]` DELETE | 删除报"草稿不存在"但 DB 里有 | 加 error 检查，区分 DB 故障和真不存在 |
| `/api/agents/[id]/chat` (5.15up bugfix) | 聊天报"组织账号已禁用"但 tenants 表里 enabled=true | `.single() → .maybeSingle()` + 加 error 检查，503 返回"加载组织信息失败" |

→ 这是个**类问题**：项目里所有 `const { data } = await db.from(...).single(); if (!data) return NOT_FOUND` 模式都可能有这 bug。后续遇到逐个修。

## 十一、API 管理模块 PR-3 · 数据迁移脚本

新建 `scripts/migrate-agent-keys-to-providers.mjs` —— 把现存正式智能体自带的
`api_key_enc` 归拢成命名 API（`model_providers`），回填 `agents.provider_id`。
**纯脚本，不改应用代码、不改 schema。**

### 11.1 设计

- **分组**：逐条解密 agent 的 `api_key_enc` → 算 HMAC-SHA256 fingerprint
  （不能按密文分组，AES-GCM 随机 IV 同 key 密文每次不同）。
  分组维度 = `category + platform + 有效 endpoint + key fingerprint`。
- `bot_id` / `assistant_id` / `model` 等留在 `agent.model_params`，不进分组、不丢。
- endpoint 为空时按 platform 补默认值（避免 PR-2 chat route 的 endpoint 非空校验拦截）。
- **幂等**：`provider_code` 稳定（`migrated-{cat}-{platform}-{endpointHash6}-{fp8}`）；
  已存在则复用；已回填 `provider_id` 的 agent 不重复处理（`.is(provider_id, null)` 守卫）。
- `provider_code` 已存在但内容不一致 → 冲突，中止、不写入、要求人工排查。
- 只新增 `model_providers` + 回填 `provider_id`，**不删 `api_key_enc`**（过渡期保留）。
- 日志不输出明文 key / 完整 fingerprint。
- 默认 **dry-run**（只预览不写）；加 `--apply` 才正式执行。

### 11.2 用法

```
node scripts/migrate-agent-keys-to-providers.mjs            # dry-run 预览
node scripts/migrate-agent-keys-to-providers.mjs --apply    # 正式执行
```

「无 api_key_enc 19 个」= 外链型 / 未配置的 agent，本就无 key 可迁，跳过即可。
