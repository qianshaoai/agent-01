## 4. 关键技术决策

### 4.1 删除策略：硬阻止而非级联

被引用时返回 409 + 引用清单，引导用户先去工作流解绑。理由：

- 数据库虽有 `ON DELETE SET NULL`，但级联会让工作流步骤变成"指向 NULL 的孤儿"，需要管理员二次手动修
- 硬阻止 + 引用清单 + 一键跳转，用户操作链路更短、心智负担更低

### 4.2 工作流引用查询限定当前页范围

`GET /api/admin/agents` 是分页接口（默认 50，最大 100）。实现时只查 `pageAgentIds` 范围内的 workflow_steps，不全表扫，避免数据增长后把"分页接口"的成本变成"全局"成本。

### 4.3 跨页 focus：URL 解析必须等客户端 hydrate

最初尝试用 `useState(() => parse(window.location))` lazy initializer 同步读 URL，但 `"use client"` 页面在 App Router 是 SSR + hydrate 双阶段：服务端首次执行 lazy initializer 时 `window === undefined`，返回 null；客户端 hydrate 不会重新执行 initializer，初值被钉死成 null。

正确做法（最终方案）：
1. 用 `useEffect` 解析 URL（保证客户端 hydrate 后才执行）
2. 加 `urlReady` 标志位
3. `load()` 在 `urlReady === false` 时不发请求
4. URL 解析完成后才触发 fetch，确保首次也是唯一一次 fetch 就带对的 `pageSize=100`

### 4.4 焦点高亮兼容筛选条件

focus effect 命中目标后会**主动清掉所有筛选**（搜索框 / 分类 / 可见性 / 启用状态），确保目标卡在 `filteredXxx` 列表里能渲染出来；否则即便 `setExpandedId` 设了，DOM 里没有这张卡，展开也无意义。

### 4.5 滚动时机：双 rAF + center

从 `setTimeout(50ms)` 改成 `requestAnimationFrame(() => requestAnimationFrame(...))`：等"清筛选 + 展开"两帧 commit 完成后再调用 `scrollIntoView`，避免高度还没变就滚到错位置。块定位由 `block:"start"` 改 `"center"`，避免被 sticky 顶栏遮挡。
