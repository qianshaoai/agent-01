# 4.28up · Phase 1 · 体验版多平台多轮上下文支持

**日期**：2026-04-29
**主分支**：master / devA+
**主题**：体验版接入第二个平台（腾讯元器）+ 修补多平台共有的两个短板（多轮上下文 + 历史回放）

---

## 1. 背景

4.28up 体验版上线时只对接了 Coze。Coze 自带 server-side conversation 持久化（`auto_save_history` + `conversation_id`），所以多轮上下文和历史回放都靠 Coze 的能力。

接入元器（腾讯）后发现：
- 元器**没有平台侧 conversation 持久化**，每次调用是独立的，AI 不记得上次聊了啥
- 元器接口中**没有 `/v1/conversation/message/list` 这种历史拉取接口**

如果继续依赖各平台的 conversation 机制，每加一个平台就要重新做一遍。这次彻底解决。

---

## 2. 核心改造

把消息历史的存储从「依赖各平台」改成「自己存一份在 DB 里」：

| 项 | 改前 | 改后 |
|----|------|------|
| 多轮上下文（非 Coze 平台）| 不工作（每次新对话）| 从本地 DB 拉历史拼成 messages 数组发给 adapter |
| 多轮上下文（Coze）| 走 Coze conversation_id + auto_save_history | 维持原状，**额外**也保存到本地 DB |
| 历史回放 | 调 Coze `/v1/conversation/message/list` | 优先从本地 DB 读；本地无数据时回退到 Coze（向后兼容旧 chat）|

**统一入口的好处**：

- 所有平台行为一致，未来加任何平台只要 adapter 跑通就行
- 历史回放不依赖外部 API，更快、更稳
- 旧 Coze chat 仍可以正常拉历史（fallback 路径保留）

## 3. 数据变更

### 3.1 新增表 `trial_messages`

```sql
CREATE TABLE trial_messages (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  chat_id     UUID NOT NULL REFERENCES trial_conversations(id) ON DELETE CASCADE,
  role        TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content     TEXT NOT NULL DEFAULT '',
  attachments JSONB,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX trial_messages_chat_idx ON trial_messages (chat_id, created_at);
```

文件：[supabase/migration_trial_messages.sql](../../supabase/migration_trial_messages.sql)

要点：
- `chat_id` 外键到 `trial_conversations`，删聊天会自动级联删消息
- `role` 只允许 `user / assistant`
- `attachments` 用 JSONB 存附件数组，结构与前端 `Attachment` 类型对齐

## 4. 接口变更

### 4.1 `POST /api/trial/chat`

新增逻辑（按 platform 分支 + 双向落库）：

```text
1. 如果 platform !== "coze"：从 trial_messages 拉历史 → 拼 messages 数组发给 adapter
2. 流式期间累计 assistantContent
3. finally：
   - 入库 user 消息（含 attachments JSONB）
   - 入库 assistant 消息（content 用累计的 assistantContent）
   - 即使 abort，partial 内容也保留入库
4. 新建 chat 立刻就 fail 且无 partial → 删整条 chat 行（避免空记录）
```

### 4.2 `GET /api/trial/conversations/[id]/messages`

读取顺序：

```text
1. 先查本地 trial_messages 表 → 有就返回，结束
2. 本地无数据 + coze_conversation_id == null → 返回空 messages
3. 本地无数据 + 有 coze_conversation_id（旧 chat 兼容）→ 调 Coze API 拉
```

## 5. 平台行为对比表

| 行为 | Coze | 元器 / 其他 |
|------|------|------------|
| 平台侧 conversation_id | 有，写入 trial_conversations.coze_conversation_id | 无（暂用 NULL）|
| messages 数组发给 adapter | 仅当前消息（Coze 自己记上下文）| 全量本地历史 + 当前消息 |
| 历史回放 | 本地 DB 优先，无则 Coze API 兜底 | 全量从本地 DB |
| 多轮上下文 | ✅ Coze 服务端 + 本地 DB 双备份 | ✅ 通过本地 DB 历史驱动 |
| 文件上传 | ✅ 走 Coze `/v1/files/upload` | ❌ 暂不支持（Phase 2 处理）|
| 中断保留部分内容 | ✅ | ✅ |
| 退出后历史可恢复 | ✅ | ✅ |
