## 二、DB 变更

### 文件：`supabase/migration_trial.sql`

操作：

1. 把 `users_user_type_check` 约束扩展为 `IN ('personal', 'organization', 'trial')`
2. UPSERT 体验账号 `18700000008 / trial001 / 12345678`
   - `tenant_code = 'PERSONAL'`
   - `user_type = 'trial'`
   - `first_login = false`（避免首次改密弹窗截断 `/trial` 跳转）
   - `pwd_hash` 为 bcrypt 12 轮 hash（密码 `12345678`）
   - `ON CONFLICT (phone, tenant_code) DO UPDATE` 强制覆盖 `username/pwd_hash/real_name/role/user_type/status/first_login`，确保即使该手机号 + PERSONAL 已存在也能干净落成体验账号

执行方式：在 Supabase SQL Editor 直接粘贴执行；执行前先确认 `18700000008+PERSONAL` 不属于正式用户。

## 四、修改的现有文件

### 4.1 身份与守卫

| 文件 | 改动 |
|------|------|
| `lib/auth.ts` | `UserPayload.userType` 类型由 `"personal" \| "organization"` 扩为加入 `"trial"`；新增 `requireFullUser(payload)`、`requireTrialUser(payload)` 两个 NextResponse-based guard helper |

### 4.2 登录与路由分流

| 文件 | 改动 |
|------|------|
| `app/api/auth/login/route.ts` | 登录响应体新增 `userType` 字段 |
| `app/login/page.tsx` | 登录成功后若 `data.userType === "trial"` 直接 `router.push("/trial")`，分支放在 `firstLogin` 判断**之前**（双保险） |
| `middleware.ts` | `USER_PROTECTED` 列表加入 `/trial`；trial 用户访问 USER_PROTECTED 路径且非 `/trial` 时 server-side 307 重定向到 `/trial`，零闪烁 |

### 4.3 正式接口 trial 守卫覆盖（共 17 个文件加 `requireFullUser` 或等价 inline 判断）

| 文件 | 守卫方式 |
|------|---------|
| `app/api/agents/route.ts` | requireFullUser |
| `app/api/agents/[id]/route.ts` | requireFullUser |
| `app/api/agents/[id]/chat/route.ts` | requireFullUser |
| `app/api/conversations/route.ts` | requireFullUser |
| `app/api/conversations/[id]/route.ts` | requireFullUser |
| `app/api/conversations/[id]/messages/route.ts` | requireFullUser |
| `app/api/upload/route.ts` | requireFullUser |
| `app/api/me/route.ts` | inline `user.userType === "trial"` 拦截 |
| `app/api/me/profile/route.ts` | requireFullUser |
| `app/api/workflows/route.ts` | requireFullUser |
| `app/api/workflows/[id]/route.ts` | requireFullUser |
| `app/api/user-agents/route.ts` | requireFullUser |
| `app/api/user-agents/[id]/route.ts` | requireFullUser |
| `app/api/user-agents/[id]/chat/route.ts` | requireFullUser |
| `app/api/speech/route.ts` | requireFullUser |
| `app/api/auth/change-password/route.ts` | requireFullUser（trial 不允许改密） |
| `app/api/auth/cancel-account/route.ts` | requireFullUser（trial 绝不允许注销） |

**未改的接口及原因**：

- `app/api/auth/login`、`logout`、`register`：trial 必须能登 / 登出
- `app/api/admin/*`：走 admin JWT，user JWT（含 trial）天然 401
- `app/api/trial/*`：体验版自己的接口，用 `requireTrialUser`
- `app/api/settings`：公共站点设置，未登录也能访问

### 4.4 配置文件

| 文件 | 改动 |
|------|------|
| `.env.local.example` | 新增占位 `TRIAL_AGENT_001_BOT_ID=` / `TRIAL_AGENT_001_API_TOKEN=` |
| `.env.local`（本地） | 已填入实际 Coze Bot ID 和 API Token（不进 Git） |

## 五、安全边界

```text
┌──────────────────────────────────────────────────────────────┐
│  trial 账号 (userType='trial')                                │
│                                                                │
│  ✓ 可访问：  /login  /logout  /trial  /api/trial/*            │
│  ✗ 被拦截：  /、/agents、/settings 等正式页面 → 重定向 /trial │
│  ✗ 被 403：  /api/agents/*、/api/conversations/*、            │
│             /api/me、/api/me/profile、/api/upload、           │
│             /api/workflows/*、/api/user-agents/*、            │
│             /api/speech、/api/auth/change-password、          │
│             /api/auth/cancel-account                          │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│  正式账号 (userType='personal' 或 'organization')             │
│                                                                │
│  ✓ 行为完全不变（所有 guard 仅对 userType==='trial' 生效）    │
│  ✗ 仍可访问 /trial 页面，但 /api/trial/* 返回 403             │
└──────────────────────────────────────────────────────────────┘
```

双层兜底：

- 前端层：middleware server-side redirect（不会闪现正式页面）
- 后端层：每个正式接口入口 `requireFullUser`（即使绕开前端直接 curl 也 403）

## 九、回滚方案

如果需要紧急回滚到 `master`：

```bash
git checkout master            # 切回正式分支即可
```

DB 侧不需要立刻回滚 migration —— `user_type` CHECK 扩展是向前兼容的（`'trial'` 取值不会污染 personal/organization 用户）。如果一定要清理：

```sql
DELETE FROM users WHERE phone = '18700000008' AND user_type = 'trial';
ALTER TABLE users DROP CONSTRAINT IF EXISTS users_user_type_check;
ALTER TABLE users ADD CONSTRAINT users_user_type_check
  CHECK (user_type IN ('personal', 'organization'));
```

环境变量 `TRIAL_AGENT_001_*` 可以保留在 `.env.local`，不会被读取，不会有副作用。
