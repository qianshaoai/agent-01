## 3. 实施要点

### 3.1 Migration 内容

```sql
-- 1) 扩展 user_type CHECK 约束以接纳 'trial'
ALTER TABLE users DROP CONSTRAINT IF EXISTS users_user_type_check;
ALTER TABLE users
  ADD CONSTRAINT users_user_type_check
  CHECK (user_type IN ('personal', 'organization', 'trial'));

-- 2) 预置体验账号（密码 12345678 的 bcrypt hash 由开发者本地生成）
-- 注意：pwd_hash 必须用 bcrypt 12 轮生成
-- users 表的唯一约束是复合键 UNIQUE (phone, tenant_code)（schema.sql:31）和 UNIQUE username
INSERT INTO users (phone, username, pwd_hash, real_name, tenant_code, user_type, status, first_login, role)
VALUES (
  '18700000008',
  'trial001',
  '<BCRYPT_HASH_OF_12345678>',
  '体验账号',
  'PERSONAL',
  'trial',
  'active',
  false,        -- 关键：避免首次改密弹窗截断 /trial 跳转
  'user'
)
ON CONFLICT (phone, tenant_code) DO UPDATE SET
  user_type   = 'trial',
  first_login = false,
  status      = 'active';
```

**手机号唯一性约束（必须确认）**：

```text
现有登录逻辑（app/api/auth/login/route.ts:36-56）允许 phone 跨 tenant_code 重复，
但当 phone 在 users 表中存在多条记录时会要求"该手机号存在多个账号，请使用用户名登录"。

为避免体验账号登录被误拦，必须满足以下任一：

A) 手机号 18700000008 全库唯一（推荐）
   动工前跑：SELECT phone, tenant_code, user_type FROM users WHERE phone='18700000008';
   - 0 行：直接 INSERT 即可
   - 1 行且 tenant_code='PERSONAL'：UPSERT 即可
   - 1 行但 tenant_code 不是 PERSONAL：先在 SQL Editor 把那一条删除或改 phone，再 INSERT
   - 多行：必须先合并/删除其余条目，否则 phone 不全库唯一

B) 体验账号始终用用户名 trial001 登录
   告知业务方：登录页 identifier 字段填 trial001，不要填手机号。
   这种情况下 phone 是否冲突无所谓，但需要在文档里明确写"用 trial001 登录"。

推荐 A 方案，登录入口对用户更友好。
```

**bcrypt hash 生成方式**（一次性，开发者本地执行）：

```bash
node -e "console.log(require('bcryptjs').hashSync('12345678', 12))"
```

把输出贴进 SQL 替换 `<BCRYPT_HASH_OF_12345678>`。

### 3.3 lib/auth.ts 新增 helper

**第 1 步**：在 `lib/auth.ts` 顶部现有 import 中合并 `NextResponse`（不要在文件末尾追加 import，TypeScript 不允许）：

```ts
// 现状（约 line 1，按实际文件为准）
import { NextRequest } from "next/server";

// 改为
import { NextRequest, NextResponse } from "next/server";
```

如果当前 import 已经包含 `NextResponse`，跳过这一步。

**第 2 步**：在 `lib/auth.ts` 文件**末尾**追加两个 helper 函数（不是 import，函数可以放末尾）：

```ts
/** 拦截体验账号访问正式业务接口 */
export function requireFullUser(payload: TokenPayload | null) {
  if (!payload || payload.type !== "user") {
    return NextResponse.json({ error: "未登录", code: "UNAUTHORIZED" }, { status: 401 });
  }
  if (payload.userType === "trial") {
    return NextResponse.json({ error: "体验账号不可调用此接口", code: "FORBIDDEN" }, { status: 403 });
  }
  return null;
}

/** 拦截非体验账号访问体验版接口 */
export function requireTrialUser(payload: TokenPayload | null) {
  if (!payload || payload.type !== "user") {
    return NextResponse.json({ error: "未登录", code: "UNAUTHORIZED" }, { status: 401 });
  }
  if (payload.userType !== "trial") {
    return NextResponse.json({ error: "非体验账号不可调用此接口", code: "FORBIDDEN" }, { status: 403 });
  }
  return null;
}
```

返回 `NextResponse | null` 的约定：null 表示通过；非空表示拦截响应，调用方直接 `return guardResp`。

调用方典型用法（阶段二 / 四会用到）：

```ts
const payload = await getPayloadFromRequest(req);
const guard = requireTrialUser(payload);
if (guard) return guard;
// ... 业务代码
```
